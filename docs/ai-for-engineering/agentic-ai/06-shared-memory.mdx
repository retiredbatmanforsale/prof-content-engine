Lesson 5's parallel executors ran in complete isolation — each started with no knowledge of what others had already retrieved. Two executors needing the same documents searched independently, fetched the same results twice, and wrote separate local copies. Worse: if two searches returned slightly different phrasings of the same fact, the synthesiser received contradictory inputs with no way to reconcile them.

**Shared memory** fixes this. Before searching, each executor asks: "has anyone already fetched this?" If yes, use the cached result. If no, search and write the result so others can benefit. Every executor reads from the same store — one consistent source of truth instead of N isolated caches.

## The race condition you must prevent

Two coroutines check `memory.get("AlphaCode benchmark")` at the same instant. Both see `None`. Both decide to search. Both fetch. Both write. The dedup failed — not because the logic was wrong, but because the check and the write weren't atomic.

```
Without a lock:
  Executor A:  get("AlphaCode") → None
  Executor B:  get("AlphaCode") → None   ← both see None before either writes
  Executor A:  search() → results
  Executor B:  search() → results         ← duplicate fetch
  Executor A:  set("AlphaCode", results)
  Executor B:  set("AlphaCode", results)  ← duplicate write
```

An `asyncio.Lock` makes the check-and-write atomic: while one coroutine holds the lock, no other can enter the critical section.

```
With asyncio.Lock:
  Executor A:  acquires lock
  Executor A:  get("AlphaCode") → None → search → set → releases lock
  Executor B:  acquires lock
  Executor B:  get("AlphaCode") → [results]  ← cache hit, no search
  Executor B:  releases lock
```

## SharedMemory design

```python
class SharedMemory:
    def __init__(self):
        self._store = {}          # {search_query: [doc_strings]}
        self._lock  = asyncio.Lock()

    async def get(self, key: str):
        async with self._lock:
            return self._store.get(key)   # None if not cached

    async def set(self, key: str, value: list) -> None:
        async with self._lock:
            self._store[key] = value

    def snapshot(self) -> dict:           # for verbose trace — no lock needed (read-only copy)
        return dict(self._store)
```

The `async with self._lock` block in `get` and `set` ensures only one coroutine runs the body at a time. `snapshot()` is called after all executors finish, so no lock is needed.

## The updated executor pattern

```
Before (L5):  search always
After  (L6):  check memory → if hit, use cache  |  if miss, search + write
```

```python
cached = await memory.get(search_query)
if cached is not None:
    results = cached           # free — no search call
else:
    results = search_tool.search(search_query)
    await memory.set(search_query, results)
```

`run()` creates one `SharedMemory` instance and passes it into every `_react()` coroutine via `asyncio.gather`.

## What shared memory did. What it didn't.

Shared memory eliminated duplicate retrieval and guaranteed a consistent source of truth for every executor. That is a real win.

But read sub-question 3's finding, then read the final answer. The finding says "No information retrieved." The final answer claims AlphaCode's approach was adopted in StarCoder and CodeLlama. That sentence was not in any retrieved document. The synthesiser invented it — and shared memory had no opinion about it.

The fix is a **Critic** — an agent that reads the synthesised answer and explicitly validates it against the findings. It flags unsupported claims, gaps, and inconsistencies. Its job is not to find a better answer, but to surface what the answer is getting wrong.
