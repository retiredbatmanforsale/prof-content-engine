## Convolution: Mathematical Recap

Let&apos;s define the operation **formally**.

Let:

<div className="not-prose my-6 rounded-xl border border-teal-200 bg-gradient-to-br from-teal-50/80 to-white p-5 dark:border-teal-900/50 dark:from-teal-950/25 dark:to-slate-900/40">
  <ul className="m-0 list-none space-y-3 p-0 text-sm text-slate-800 dark:text-slate-200">
    <li className="flex flex-wrap items-baseline gap-2">
      <span className="font-mono font-bold text-teal-800 dark:text-teal-300">I ∈ ℝ<sup>H×W</sup>:</span>
      <span>Input image (grayscale)</span>
    </li>
    <li className="flex flex-wrap items-baseline gap-2">
      <span className="font-mono font-bold text-teal-800 dark:text-teal-300">F ∈ ℝ<sup>k×k</sup>:</span>
      <span>Filter of size <em>k</em>×<em>k</em></span>
    </li>
    <li className="flex flex-wrap items-baseline gap-2">
      <span className="font-mono font-bold text-teal-800 dark:text-teal-300">O ∈ ℝ<sup>H′×W′</sup>:</span>
      <span>Output feature map</span>
    </li>
  </ul>
</div>

Then the **convolution** operation at location **(i, j)** is defined as:

<div className="not-prose my-6 flex flex-col items-center gap-4 rounded-xl border-2 border-dashed border-violet-300 bg-violet-50/40 p-5 dark:border-violet-800 dark:bg-violet-950/20">
  <p className="m-0 text-center text-xs font-semibold uppercase tracking-wide text-violet-800 dark:text-violet-300">One output pixel</p>
  <div className="flex flex-wrap items-center justify-center gap-3 text-sm text-slate-700 dark:text-slate-300">
    <span className="rounded-lg border-2 border-slate-400 bg-white px-3 py-2 font-mono text-xs dark:border-slate-500 dark:bg-slate-800">patch of <strong>I</strong></span>
    <span className="text-violet-600 dark:text-violet-400">⊙</span>
    <span className="rounded-lg border-2 border-slate-400 bg-white px-3 py-2 font-mono text-xs dark:border-slate-500 dark:bg-slate-800"><strong>F</strong></span>
    <span className="text-violet-600 dark:text-violet-400">→</span>
    <span className="rounded-lg border-2 border-violet-500 bg-violet-100 px-3 py-2 font-mono text-xs font-bold dark:border-violet-600 dark:bg-violet-950/50">sum → scalar</span>
  </div>
</div>

This is an **element-wise multiplication** of the filter and the **patch** of the image it covers.

The result is **summed** up to a **single scalar**.

We then **slide** this filter across the image (usually with **stride = 1**).

Optionally, we add a **bias** and apply a **nonlinearity** (e.g., **ReLU**).

This gives us a **2D output** — a **feature map** — indicating **where** in the image that particular pattern is **detected**.

## Non-linearity: ReLU

After the convolution (and an optional bias), we apply a **non-linearity** — most commonly **ReLU** (Rectified Linear Unit): `ReLU(x) = max(0, x)`.

It's stupidly simple — **negative values get clipped to zero**, positive values pass through unchanged:

<div className="not-prose my-6 flex flex-wrap items-center justify-center gap-4 rounded-xl border-2 border-dashed border-sky-300 bg-sky-50/40 p-5 dark:border-sky-800 dark:bg-sky-950/20">
  <div className="flex flex-col items-center gap-2">
    <p className="m-0 text-[11px] font-bold uppercase tracking-wide text-sky-800 dark:text-sky-300">
      Feature map (raw)
    </p>
    <div className="inline-grid grid-cols-4 gap-1 rounded-lg border-2 border-slate-300 bg-white p-2 dark:border-slate-600 dark:bg-slate-800">
      {[
        [-2, 3, -1, 0],
        [4, -3, 2, 1],
        [-1, 5, -4, 2],
        [0, -2, 3, -1],
      ].map((row, ri) =>
        row.map((v, ci) => (
          <span
            key={`relu-pre-${ri}-${ci}`}
            className={`flex h-9 w-9 items-center justify-center rounded-md font-mono text-xs font-bold ${
              v < 0
                ? 'bg-rose-200 text-rose-950 dark:bg-rose-900/60 dark:text-rose-100'
                : v > 0
                  ? 'bg-emerald-200 text-emerald-950 dark:bg-emerald-900/60 dark:text-emerald-100'
                  : 'bg-slate-200 text-slate-700 dark:bg-slate-700 dark:text-slate-200'
            }`}
          >
            {v > 0 ? `+${v}` : v}
          </span>
        )),
      )}
    </div>
  </div>
  <span className="text-sm font-bold text-sky-700 dark:text-sky-400">→ ReLU →</span>
  <div className="flex flex-col items-center gap-2">
    <p className="m-0 text-[11px] font-bold uppercase tracking-wide text-sky-800 dark:text-sky-300">
      After ReLU
    </p>
    <div className="inline-grid grid-cols-4 gap-1 rounded-lg border-2 border-slate-300 bg-white p-2 dark:border-slate-600 dark:bg-slate-800">
      {[
        [0, 3, 0, 0],
        [4, 0, 2, 1],
        [0, 5, 0, 2],
        [0, 0, 3, 0],
      ].map((row, ri) =>
        row.map((v, ci) => (
          <span
            key={`relu-post-${ri}-${ci}`}
            className={`flex h-9 w-9 items-center justify-center rounded-md font-mono text-xs font-bold ${
              v > 0
                ? 'bg-emerald-200 text-emerald-950 dark:bg-emerald-900/60 dark:text-emerald-100'
                : 'bg-slate-200 text-slate-700 dark:bg-slate-700 dark:text-slate-200'
            }`}
          >
            {v > 0 ? `+${v}` : v}
          </span>
        )),
      )}
    </div>
  </div>
</div>

Why bother? Without a non-linearity, **stacking conv layers would collapse into a single linear transformation** — depth would buy nothing. ReLU also has a clean interpretation: a vertical-edge filter produces **positive** activations where its pattern is present and **negative** activations on the inverse pattern; ReLU keeps the "pattern detected" signal and discards the inverse.

## Pooling

A **pooling** layer summarizes a local neighborhood into a single value, **shrinking** the feature map. Two common variants:

- **Max pooling** — take the **maximum** of each window. Preserves the strongest activation.
- **Average pooling** — take the **mean**. Smooths the response.

The classic choice is **2×2 max pooling with stride 2** — non-overlapping 2×2 windows, **halving both H and W**:

<div className="not-prose my-6 flex flex-wrap items-center justify-center gap-4 rounded-xl border-2 border-dashed border-amber-300 bg-amber-50/40 p-5 dark:border-amber-800 dark:bg-amber-950/20">
  <div className="flex flex-col items-center gap-2">
    <p className="m-0 text-[11px] font-bold uppercase tracking-wide text-amber-800 dark:text-amber-300">
      Feature map (4×4)
    </p>
    <div className="inline-grid grid-cols-2 gap-2 rounded-lg border-2 border-slate-300 bg-white p-2 dark:border-slate-600 dark:bg-slate-800">
      {[
        {data: [[1, 3], [2, 5]], maxR: 1, maxC: 1},
        {data: [[2, 1], [0, 1]], maxR: 0, maxC: 0},
        {data: [[0, 1], [1, 4]], maxR: 1, maxC: 1},
        {data: [[6, 2], [3, 4]], maxR: 0, maxC: 0},
      ].map((win, wi) => (
        <div
          key={`pool-win-${wi}`}
          className="inline-grid grid-cols-2 gap-0.5 rounded border-2 border-amber-400 bg-amber-50 p-1 dark:border-amber-600 dark:bg-amber-950/30"
        >
          {win.data.map((row, ri) =>
            row.map((v, ci) => {
              const isMax = ri === win.maxR && ci === win.maxC;
              return (
                <span
                  key={`pool-win-${wi}-${ri}-${ci}`}
                  className={`flex h-8 w-8 items-center justify-center rounded font-mono text-xs font-bold ${
                    isMax
                      ? 'bg-amber-400 text-amber-950 ring-1 ring-amber-700 dark:bg-amber-600 dark:text-amber-50'
                      : 'bg-white text-slate-600 dark:bg-slate-800 dark:text-slate-300'
                  }`}
                >
                  {v}
                </span>
              );
            }),
          )}
        </div>
      ))}
    </div>
  </div>
  <span className="text-sm font-bold text-amber-700 dark:text-amber-400">→ 2×2 max pool →</span>
  <div className="flex flex-col items-center gap-2">
    <p className="m-0 text-[11px] font-bold uppercase tracking-wide text-amber-800 dark:text-amber-300">
      Pooled (2×2)
    </p>
    <div className="inline-grid grid-cols-2 gap-1 rounded-lg border-2 border-slate-300 bg-white p-2 dark:border-slate-600 dark:bg-slate-800">
      {[5, 2, 4, 6].map((v, i) => (
        <span
          key={`pool-out-${i}`}
          className="flex h-9 w-9 items-center justify-center rounded-md bg-amber-400 font-mono text-xs font-bold text-amber-950 ring-1 ring-amber-700 dark:bg-amber-600 dark:text-amber-50"
        >
          {v}
        </span>
      ))}
    </div>
  </div>
</div>

Why pooling helps:

- **Translation tolerance** — a small shift in the input doesn't change the output as long as the strongest activation stays inside the same window.
- **Cheaper downstream compute** — fewer cells for later layers to process.
- **Wider effective receptive fields** — neurons deep in the network end up "seeing" larger regions of the original input image.
