## Forward Pass of the CNN

The CNN&apos;s forward pass has **two distinct phases**:

1. A **feature extractor** — stacked **conv blocks** that turn pixels into rich feature maps.
2. A **classifier head** — **fully-connected (FC / FFN) layers** ending in a **softmax** that turns those features into class probabilities.

Without the FC head, you&apos;d have *feature maps* — not class predictions. **Conv blocks alone aren&apos;t a CNN.** The FC layers on the end are what turn extracted features into a usable answer.

There are <strong>three core operations</strong> inside each conv block:

<div className="not-prose my-6 space-y-4">
  <div className="rounded-xl border-2 border-sky-400 bg-gradient-to-br from-sky-50 to-white p-5 shadow-sm dark:border-sky-700 dark:from-sky-950/40 dark:to-slate-900/40">
    <p className="m-0 text-sm font-bold text-sky-900 dark:text-sky-200">(i) Convolution</p>
    <ul className="mt-3 mb-0 list-disc space-y-2 pl-5 text-sm text-slate-800 dark:text-slate-200">
      <li>Captures <strong>local features</strong>.</li>
      <li>Each filter learns to detect a <strong>specific pattern</strong> (edge, texture, shape).</li>
      <li>Produces <strong>multiple feature maps</strong>, each indicating <em>where</em> that pattern appears.</li>
    </ul>
  </div>
  <div className="rounded-xl border-2 border-amber-400 bg-gradient-to-br from-amber-50 to-white p-5 shadow-sm dark:border-amber-700 dark:from-amber-950/40 dark:to-slate-900/40">
    <p className="m-0 text-sm font-bold text-amber-900 dark:text-amber-200">
      (ii) ReLU: Nonlinearity (e.g., ReLU) Rectified Linear Unit
    </p>
    <ul className="mt-3 mb-0 list-disc space-y-2 pl-5 text-sm text-slate-800 dark:text-slate-200">
      <li>Applies an <strong>activation function</strong> to the feature map output.</li>
      <li>
        ReLU sets all <strong>negative values to zero</strong> (like a <em>threshold gate</em>).
      </li>
      <li>
        This nonlinearity makes the network <strong>expressive</strong>, i.e., capable of learning <em>complex, nonlinear</em> patterns.
      </li>
    </ul>
  </div>
  <div className="rounded-xl border-2 border-violet-400 bg-gradient-to-br from-violet-50 to-white p-5 shadow-sm dark:border-violet-700 dark:from-violet-950/40 dark:to-slate-900/40">
    <p className="m-0 text-sm font-bold text-violet-900 dark:text-violet-200">(iii) Pooling (Downsampling)</p>
    <ul className="mt-3 mb-0 list-disc space-y-2 pl-5 text-sm text-slate-800 dark:text-slate-200">
      <li>
        Reduces the <strong>spatial dimensions</strong> (e.g., <strong>28×28 → 14×14</strong>).
      </li>
      <li>Helps <strong>reduce computation</strong> and introduces <strong>invariance</strong> to small translations/rotations.</li>
      <li>
        <strong>Common:</strong> Max pooling, which retains only the <strong>max value</strong> in each small window.
      </li>
      <li>
        <strong>Mean Pooling:</strong> averages out the whole matrix and preserves the <strong>mean/avg</strong> instead of max.
      </li>
    </ul>
  </div>
</div>

These steps <strong>repeat across layers</strong>, allowing the network to learn <em>increasingly abstract</em> and <em>complex</em> representations. Then the **classifier head** (Flatten → FC → Softmax) turns those features into a class prediction. **Step through every stage** below to watch a toy CNN classify a vertical bar:

## Local Connectivity + Hierarchical Features

CNNs exploit <strong>local connectivity</strong>: each neuron only &quot;sees&quot; a <strong>small patch</strong> of the image. However, by <strong>stacking multiple convolution layers</strong>, the <strong>effective receptive field</strong> grows:

<div className="not-prose my-6 flex flex-col items-stretch gap-2 md:flex-row md:items-stretch">
  <div className="flex-1 rounded-xl border-2 border-cyan-300 bg-cyan-50/90 p-4 dark:border-cyan-800 dark:bg-cyan-950/25">
    <p className="m-0 text-xs font-bold uppercase tracking-wide text-cyan-900 dark:text-cyan-200">Layer 1</p>
    <p className="mt-2 mb-0 text-sm text-slate-800 dark:text-slate-200">Learns <strong>edges</strong> (horizontal, vertical, diagonal).</p>
  </div>
  <div className="hidden items-center justify-center text-cyan-500 md:flex" aria-hidden>
    →
  </div>
  <div className="flex-1 rounded-xl border-2 border-fuchsia-300 bg-fuchsia-50/90 p-4 dark:border-fuchsia-800 dark:bg-fuchsia-950/25">
    <p className="m-0 text-xs font-bold uppercase tracking-wide text-fuchsia-900 dark:text-fuchsia-200">Layer 2</p>
    <p className="mt-2 mb-0 text-sm text-slate-800 dark:text-slate-200">
      Learns <strong>combinations</strong> of those edges (<em>corners</em>, <em>curves</em>).
    </p>
  </div>
  <div className="hidden items-center justify-center text-fuchsia-500 md:flex" aria-hidden>
    →
  </div>
  <div className="flex-1 rounded-xl border-2 border-lime-300 bg-lime-50/90 p-4 dark:border-lime-800 dark:bg-lime-950/25">
    <p className="m-0 text-xs font-bold uppercase tracking-wide text-lime-900 dark:text-lime-200">Layer 3+</p>
    <p className="mt-2 mb-0 text-sm text-slate-800 dark:text-slate-200">
      Starts recognizing more <strong>semantic</strong> features (<em>eyes</em>, <em>faces</em>, <em>objects</em>).
    </p>
  </div>
</div>

This is <strong>hierarchical feature extraction</strong>, a <em>key power</em> of CNNs.

## Multiple Filters per Layer = Feature Volume

Each conv layer learns <strong>not one, but many filters</strong>—each producing its own <strong>feature map</strong>.

Stacking all of these maps together gives a <strong>feature volume</strong> (like a <strong>3D tensor</strong>).

## Training: How Filters Are Learned

Filters are **not designed** — they are **learned from data** via **gradient descent**. Three steps repeat for every training batch:

<div className="not-prose my-6 grid gap-3 md:grid-cols-3">
  <div className="rounded-xl border-2 border-sky-300 bg-sky-50/70 p-4 dark:border-sky-800 dark:bg-sky-950/25">
    <div className="flex items-center gap-2">
      <span className="flex h-7 w-7 items-center justify-center rounded-full bg-sky-600 text-xs font-bold text-white">1</span>
      <p className="m-0 text-sm font-bold text-sky-900 dark:text-sky-200">Forward pass + loss</p>
    </div>
    <p className="mt-2 mb-0 text-sm text-slate-800 dark:text-slate-200">
      Push the image through the CNN; compare prediction <span className="font-mono">ŷ</span> to the true label <span className="font-mono">y</span> via a loss <span className="font-mono">L(ŷ, y)</span> (typically <strong>cross-entropy</strong>).
    </p>
  </div>
  <div className="rounded-xl border-2 border-rose-300 bg-rose-50/70 p-4 dark:border-rose-800 dark:bg-rose-950/25">
    <div className="flex items-center gap-2">
      <span className="flex h-7 w-7 items-center justify-center rounded-full bg-rose-600 text-xs font-bold text-white">2</span>
      <p className="m-0 text-sm font-bold text-rose-900 dark:text-rose-200">Backward pass</p>
    </div>
    <p className="mt-2 mb-0 text-sm text-slate-800 dark:text-slate-200">
      Compute <span className="font-mono">∂L/∂w</span> for every weight using the <strong>chain rule</strong> — gradients flow <em>backwards</em> through the same layers used for the forward pass.
    </p>
  </div>
  <div className="rounded-xl border-2 border-emerald-300 bg-emerald-50/70 p-4 dark:border-emerald-800 dark:bg-emerald-950/25">
    <div className="flex items-center gap-2">
      <span className="flex h-7 w-7 items-center justify-center rounded-full bg-emerald-600 text-xs font-bold text-white">3</span>
      <p className="m-0 text-sm font-bold text-emerald-900 dark:text-emerald-200">Update</p>
    </div>
    <p className="mt-2 mb-0 text-sm text-slate-800 dark:text-slate-200">
      Nudge every weight a tiny step against its gradient:{' '}
      <span className="font-mono font-bold">w ← w − η · ∂L/∂w</span>, where <span className="font-mono">η</span> is the <em>learning rate</em>.
    </p>
  </div>
</div>

**Which loss?** That depends on the task — the rest of the training loop stays the same:

<div className="not-prose my-6 rounded-xl border-2 border-dashed border-slate-300 bg-slate-50/60 p-5 dark:border-slate-700 dark:bg-slate-800/30">
  <p className="m-0 text-[11px] font-bold uppercase tracking-wide text-slate-600 dark:text-slate-400">
    Common loss choices
  </p>
  <div className="mt-3 grid gap-4 sm:grid-cols-3">
    <div className="rounded-lg border border-slate-200 bg-white p-3 dark:border-slate-600 dark:bg-slate-900/50">
      <p className="m-0 text-sm font-bold text-slate-900 dark:text-slate-100">Binary cross-entropy</p>
      <p className="m-0 mt-1 text-[11px] italic text-slate-600 dark:text-slate-400">2-class (cat vs. dog)</p>
      <p className="m-0 mt-2 font-mono text-[11px] text-slate-700 dark:text-slate-300">
        L = −[y · log ŷ + (1−y) · log(1−ŷ)]
      </p>
    </div>
    <div className="rounded-lg border border-slate-200 bg-white p-3 dark:border-slate-600 dark:bg-slate-900/50">
      <p className="m-0 text-sm font-bold text-slate-900 dark:text-slate-100">Categorical cross-entropy</p>
      <p className="m-0 mt-1 text-[11px] italic text-slate-600 dark:text-slate-400">K-class with softmax outputs</p>
      <p className="m-0 mt-2 font-mono text-[11px] text-slate-700 dark:text-slate-300">
        L = −Σ<sub>k</sub> y<sub>k</sub> · log ŷ<sub>k</sub>
      </p>
    </div>
    <div className="rounded-lg border border-slate-200 bg-white p-3 dark:border-slate-600 dark:bg-slate-900/50">
      <p className="m-0 text-sm font-bold text-slate-900 dark:text-slate-100">Mean Squared Error</p>
      <p className="m-0 mt-1 text-[11px] italic text-slate-600 dark:text-slate-400">regression (predict a number)</p>
      <p className="m-0 mt-2 font-mono text-[11px] text-slate-700 dark:text-slate-300">
        L = (1/N) · Σ (y − ŷ)<sup>2</sup>
      </p>
    </div>
  </div>
  <p className="m-0 mt-3 text-[11px] italic text-slate-600 dark:text-slate-400">
    The loss shapes which gradients flow back; everything downstream — backward pass, weight update — is unchanged.
  </p>
</div>

**Gradients travel the same path — in reverse.** The forward pass produces activations layer by layer; the backward pass produces gradients layer by layer, in opposite order:

<div className="not-prose my-6 overflow-x-auto rounded-2xl border-2 border-slate-200 bg-white p-4 dark:border-slate-700 dark:bg-slate-800/30">
  <div className="flex min-w-[640px] flex-col gap-3">
    <div className="flex items-center gap-2">
      <span className="w-24 shrink-0 text-right text-[10px] font-bold uppercase tracking-wide text-teal-700 dark:text-teal-300">
        Forward →
      </span>
      <div className="flex flex-1 items-center justify-between gap-1">
        {['Input', 'Conv', 'ReLU', 'Pool', 'Conv', 'ReLU', 'FC', 'Loss'].map((label, i) => (
          <div key={`fwd-${i}`} className="flex items-center gap-1">
            <div className="rounded-md border-2 border-teal-300 bg-teal-50 px-2 py-1 text-[11px] font-semibold text-teal-900 dark:border-teal-700 dark:bg-teal-950/40 dark:text-teal-200">
              {label}
            </div>
            {i < 7 && <span className="text-sm text-teal-500" aria-hidden>→</span>}
          </div>
        ))}
      </div>
    </div>
    <div className="flex items-center gap-2">
      <span className="w-24 shrink-0 text-right text-[10px] font-bold uppercase tracking-wide text-rose-700 dark:text-rose-300">
        ← Backward
      </span>
      <div className="flex flex-1 items-center justify-between gap-1">
        {['∂L/∂x', '∂L/∂Wᶜ', '∂L/∂ReLU', '∂L/∂Pool', '∂L/∂Wᶜ', '∂L/∂ReLU', '∂L/∂W', 'L'].map((label, i) => (
          <div key={`bwd-${i}`} className="flex items-center gap-1">
            <div className="rounded-md border-2 border-rose-300 bg-rose-50 px-2 py-1 font-mono text-[11px] font-semibold text-rose-900 dark:border-rose-700 dark:bg-rose-950/40 dark:text-rose-200">
              {label}
            </div>
            {i < 7 && <span className="text-sm text-rose-500" aria-hidden>←</span>}
          </div>
        ))}
      </div>
    </div>
    <p className="m-0 mt-1 text-center text-[11px] italic text-slate-600 dark:text-slate-400">
      Chain rule: gradient at each layer is computed from the gradient at the layer <em>after</em> it.
    </p>
  </div>
</div>

Repeat this loop **millions of times** across batches of training images. Each pass nudges every filter weight in the direction that lowers the loss. Over time, filters **specialize** to detect patterns most useful for distinguishing the target classes.

## Classification via Softmax

Once you&apos;ve passed the image through multiple <strong>convolution + ReLU + pooling</strong> layers:

You&apos;re left with <strong>abstract high-level features</strong>.

<strong>Flatten</strong> them and feed them into a <strong>fully connected (dense) layer</strong>.

The <strong>final layer</strong> uses <strong>Softmax</strong> to produce a <strong>probability distribution</strong> over possible classes:

      <span className="px-2">Σ<sub>j=1</sub><sup>K</sup> e<sup>z<sub>j</sub></sup></span>
    </span>

  <p className="m-0 text-xs text-slate-600 dark:text-slate-400">
