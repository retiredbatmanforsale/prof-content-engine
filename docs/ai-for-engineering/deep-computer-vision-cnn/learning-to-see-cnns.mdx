## From Convolution to CNN: Zooming Out

You already know the basic idea:

<div className="not-prose my-6 grid gap-3 md:grid-cols-2">
  <div className="rounded-xl border-2 border-teal-300 bg-teal-50/90 p-4 dark:border-teal-800 dark:bg-teal-950/30">
    <p className="m-0 text-xs font-bold uppercase tracking-wide text-teal-800 dark:text-teal-300">Input</p>
    <p className="mt-2 mb-0 text-sm text-slate-800 dark:text-slate-200">The <strong>image</strong> is input.</p>
  </div>
  <div className="rounded-xl border-2 border-indigo-300 bg-indigo-50/90 p-4 dark:border-indigo-800 dark:bg-indigo-950/30">
    <p className="m-0 text-xs font-bold uppercase tracking-wide text-indigo-800 dark:text-indigo-300">Convolution</p>
    <p className="mt-2 mb-0 text-sm text-slate-800 dark:text-slate-200">
      The <strong>filters</strong> (also called <strong>kernels</strong>) are <em>slid</em> over the image to compute <strong>feature maps</strong> via convolution.
    </p>
  </div>
</div>

The resulting numbers capture the <strong>presence</strong> (or <strong>absence</strong>) of specific patterns (<em>edges</em>, <em>corners</em>, etc.) in the image.

But so far, we <strong>assumed</strong> we had those filters. In real CNNs:

<div className="not-prose my-5 rounded-xl border-2 border-dashed border-emerald-400 bg-emerald-50/70 p-4 text-center dark:border-emerald-700 dark:bg-emerald-950/30">
  <p className="m-0 text-base font-semibold text-emerald-900 dark:text-emerald-200">We <strong>learn</strong> those filters <strong>automatically from data</strong>.</p>
</div>

## The Core Components of a CNN

There are <strong>three core operations</strong> you&apos;ll repeatedly see in CNNs:

<div className="not-prose my-6 space-y-4">
  <div className="rounded-xl border-2 border-sky-400 bg-gradient-to-br from-sky-50 to-white p-5 shadow-sm dark:border-sky-700 dark:from-sky-950/40 dark:to-slate-900/40">
    <p className="m-0 text-sm font-bold text-sky-900 dark:text-sky-200">(i) Convolution</p>
    <ul className="mt-3 mb-0 list-disc space-y-2 pl-5 text-sm text-slate-800 dark:text-slate-200">
      <li>Captures <strong>local features</strong>.</li>
      <li>Each filter learns to detect a <strong>specific pattern</strong> (edge, texture, shape).</li>
      <li>Produces <strong>multiple feature maps</strong>, each indicating <em>where</em> that pattern appears.</li>
    </ul>
  </div>
  <div className="rounded-xl border-2 border-amber-400 bg-gradient-to-br from-amber-50 to-white p-5 shadow-sm dark:border-amber-700 dark:from-amber-950/40 dark:to-slate-900/40">
    <p className="m-0 text-sm font-bold text-amber-900 dark:text-amber-200">
      (ii) ReLU: Nonlinearity (e.g., ReLU) Rectified Linear Unit
    </p>
    <ul className="mt-3 mb-0 list-disc space-y-2 pl-5 text-sm text-slate-800 dark:text-slate-200">
      <li>Applies an <strong>activation function</strong> to the feature map output.</li>
      <li>
        ReLU sets all <strong>negative values to zero</strong> (like a <em>threshold gate</em>).
      </li>
      <li>
        This nonlinearity makes the network <strong>expressive</strong>, i.e., capable of learning <em>complex, nonlinear</em> patterns.
      </li>
    </ul>
  </div>
  <div className="not-prose overflow-hidden rounded-xl border-2 border-amber-300 bg-white shadow-sm dark:border-amber-700 dark:bg-slate-900">
    <img
      src={nonLinearityReluLesson8}
      alt="Illustration of non-linearity with ReLU on feature map values"
      className="mx-auto block h-auto w-full max-w-2xl object-contain p-2"
      loading="lazy"
      decoding="async"
    />
  </div>
  <div className="rounded-xl border-2 border-violet-400 bg-gradient-to-br from-violet-50 to-white p-5 shadow-sm dark:border-violet-700 dark:from-violet-950/40 dark:to-slate-900/40">
    <p className="m-0 text-sm font-bold text-violet-900 dark:text-violet-200">(iii) Pooling (Downsampling)</p>
    <ul className="mt-3 mb-0 list-disc space-y-2 pl-5 text-sm text-slate-800 dark:text-slate-200">
      <li>
        Reduces the <strong>spatial dimensions</strong> (e.g., <strong>28×28 → 14×14</strong>).
      </li>
      <li>Helps <strong>reduce computation</strong> and introduces <strong>invariance</strong> to small translations/rotations.</li>
      <li>
        <strong>Common:</strong> Max pooling, which retains only the <strong>max value</strong> in each small window.
      </li>
      <li>
        <strong>Mean Pooling:</strong> averages out the whole matrix and preserves the <strong>mean/avg</strong> instead of max.
      </li>
    </ul>
  </div>
</div>

These steps <strong>repeat across layers</strong>, allowing the network to learn <em>increasingly abstract</em> and <em>complex</em> representations.

## Local Connectivity + Hierarchical Features

CNNs exploit <strong>local connectivity</strong>: each neuron only &quot;sees&quot; a <strong>small patch</strong> of the image. However, by <strong>stacking multiple convolution layers</strong>, the <strong>effective receptive field</strong> grows:

<div className="not-prose my-6 flex flex-col items-stretch gap-2 md:flex-row md:items-stretch">
  <div className="flex-1 rounded-xl border-2 border-cyan-300 bg-cyan-50/90 p-4 dark:border-cyan-800 dark:bg-cyan-950/25">
    <p className="m-0 text-xs font-bold uppercase tracking-wide text-cyan-900 dark:text-cyan-200">Layer 1</p>
    <p className="mt-2 mb-0 text-sm text-slate-800 dark:text-slate-200">Learns <strong>edges</strong> (horizontal, vertical, diagonal).</p>
  </div>
  <div className="hidden items-center justify-center text-cyan-500 md:flex" aria-hidden>
    →
  </div>
  <div className="flex-1 rounded-xl border-2 border-fuchsia-300 bg-fuchsia-50/90 p-4 dark:border-fuchsia-800 dark:bg-fuchsia-950/25">
    <p className="m-0 text-xs font-bold uppercase tracking-wide text-fuchsia-900 dark:text-fuchsia-200">Layer 2</p>
    <p className="mt-2 mb-0 text-sm text-slate-800 dark:text-slate-200">
      Learns <strong>combinations</strong> of those edges (<em>corners</em>, <em>curves</em>).
    </p>
  </div>
  <div className="hidden items-center justify-center text-fuchsia-500 md:flex" aria-hidden>
    →
  </div>
  <div className="flex-1 rounded-xl border-2 border-lime-300 bg-lime-50/90 p-4 dark:border-lime-800 dark:bg-lime-950/25">
    <p className="m-0 text-xs font-bold uppercase tracking-wide text-lime-900 dark:text-lime-200">Layer 3+</p>
    <p className="mt-2 mb-0 text-sm text-slate-800 dark:text-slate-200">
      Starts recognizing more <strong>semantic</strong> features (<em>eyes</em>, <em>faces</em>, <em>objects</em>).
    </p>
  </div>
</div>

This is <strong>hierarchical feature extraction</strong>, a <em>key power</em> of CNNs.

## Multiple Filters per Layer = Feature Volume

Each conv layer learns <strong>not one, but many filters</strong>—each producing its own <strong>feature map</strong>.

Stacking all of these maps together gives a <strong>feature volume</strong> (like a <strong>3D tensor</strong>).

## Classification via Softmax

Once you&apos;ve passed the image through multiple <strong>convolution + ReLU + pooling</strong> layers:

You&apos;re left with <strong>abstract high-level features</strong>.

<strong>Flatten</strong> them and feed them into a <strong>fully connected (dense) layer</strong>.

The <strong>final layer</strong> uses <strong>Softmax</strong> to produce a <strong>probability distribution</strong> over possible classes.

**Softmax formula:**

$$
\text{Softmax}(z_i) = \frac{e^{z_i}}{\sum_{j=1}^{K} e^{z_j}}
$$

<div className="not-prose my-5 rounded-xl border border-slate-200 bg-slate-50 p-4 dark:border-slate-600 dark:bg-slate-900/50">
  <p className="m-0 text-xs font-bold uppercase tracking-wide text-slate-600 dark:text-slate-400">Where</p>
  <ul className="mt-2 mb-0 list-disc space-y-1 pl-5 text-sm text-slate-800 dark:text-slate-200">
    <li>
      <span className="font-mono font-semibold">z<sub>i</sub></span> is the score for class <em>i</em>
    </li>
    <li>
      <span className="font-mono font-semibold">K</span> is the total number of classes
    </li>
  </ul>
</div>

**This ensures:**

<div className="not-prose my-4 grid gap-3 sm:grid-cols-2">
  <div className="rounded-lg border-2 border-emerald-300 bg-emerald-50/80 p-3 text-center text-sm font-medium text-emerald-900 dark:border-emerald-800 dark:bg-emerald-950/30 dark:text-emerald-200">
    Each output is in <strong>[0, 1]</strong>
  </div>
  <div className="rounded-lg border-2 border-emerald-300 bg-emerald-50/80 p-3 text-center text-sm font-medium text-emerald-900 dark:border-emerald-800 dark:bg-emerald-950/30 dark:text-emerald-200">
    All outputs <strong>sum to 1</strong> (so they form a valid <em>probability distribution</em>)
  </div>
</div>
