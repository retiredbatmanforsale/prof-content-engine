## The limits of fully connected networks in vision

We have explored **fully connected neural networks** models where **each neuron in one layer is connected to every neuron in the previous layer**. This structure works well for problems where the input is **one-dimensional** or where **SPATIAL LOCALITY** is *not* important.

But **images are fundamentally different**.

An image is a **two-dimensional structure** — width and height (and possibly **channels** for color). Every pixel has **SPATIAL meaning**: a pixel next to another shares **more visual relevance** than one on the *opposite side* of the image. **Flattening** this structure into a **one-dimensional vector** *destroys this locality* — and the parameter count explodes (the interactive in [Lesson 4](./feature-detection-hierarchy#deep-learning-as-the-paradigm-shift) lets you watch that happen directly).

## What we need instead

We need an architecture that <span className="rounded bg-rose-100 px-1 py-0.5 dark:bg-rose-950/50"><Term def="Keeps the image as a 2D grid throughout the network instead of flattening it into a 1D vector. Neighbor relationships (left/right/up/down) survive — every output position still corresponds to a region in the input."><strong>preserves spatial layout</strong></Term></span>, has <span className="rounded bg-rose-100 px-1 py-0.5 dark:bg-rose-950/50"><Term def="Each output neuron looks only at a small nearby region of the input (its receptive field), not every pixel. Mirrors how nearby pixels carry the most relevant visual signal."><strong>local connectivity</strong></Term></span>, is parameter-efficient via <span className="rounded bg-rose-100 px-1 py-0.5 dark:bg-rose-950/50"><Term def="The same small set of filter weights is reused at every spatial position. A handful of weights (e.g. 9 for a 3×3 filter) replaces millions of fully-connected weights — drastically reducing the parameter budget."><strong>weight sharing</strong></Term></span>, and naturally <span className="rounded bg-rose-100 px-1 py-0.5 dark:bg-rose-950/50"><Term def="Lower layers detect simple primitives (edges, color blobs); middle layers compose them into parts (eyes, wheels); higher layers assemble whole objects. Each level builds on the one below."><strong>builds hierarchies of features</strong></Term></span>. That's the idea behind **Convolutional Neural Networks (CNNs)**.

## What is a convolution?

A **convolution** is a **sliding**, **element-wise multiplication** operation followed by a **sum**, **bias** addition, and **nonlinearity**.

Here&apos;s how it works:

Watch the operation play out: a **3×3 kernel** slides across a small image, multiplies element-wise with each patch, sums the result, and writes that single number into the **feature map**. Try different kernels — the same image produces different feature maps depending on what the kernel is tuned to detect.

The result is a new matrix, called a **feature map**, which encodes **where** and **how strongly** a certain pattern appears in the image.

By **stacking multiple filters**, we extract **multiple types of features** — *edges in different directions*, *textures*, *corners*, etc.

## Local connectivity and weight sharing

Instead of connecting **every input pixel** to **every neuron** in the next layer, CNNs use **local receptive fields** (**KERNELS**) in **small patches** of the input image.

Let&apos;s say we use a small **filter** (also called a **kernel**) of size **3×3**. This filter is a tiny set of weights (**just 9 numbers**) that will be applied across the image:

  <p className="m-0 text-center text-xs font-semibold uppercase tracking-wide text-slate-600 dark:text-slate-400">Sliding the same 3×3 filter</p>
  <div className="mt-4 flex flex-col items-center gap-3 text-sm text-slate-800 dark:text-slate-200">
    <div className="flex flex-wrap items-center justify-center gap-2">
      <span className="rounded-lg border-2 border-teal-500 bg-white px-3 py-2 font-mono text-xs shadow-sm dark:bg-slate-800">Top-left 3×3 patch</span>
      <span className="text-teal-600 dark:text-teal-400">→</span>
      <span className="rounded-lg border-2 border-teal-600 bg-teal-100 px-3 py-2 text-xs font-semibold dark:border-teal-500 dark:bg-teal-950/50">1st output neuron</span>
    </div>
    <span className="text-slate-400" aria-hidden>
      ↓ shift 1 pixel right
    </span>
    <div className="flex flex-wrap items-center justify-center gap-2">
      <span className="rounded-lg border-2 border-teal-500 bg-white px-3 py-2 font-mono text-xs shadow-sm dark:bg-slate-800">Next 3×3 patch</span>
      <span className="text-teal-600 dark:text-teal-400">→</span>
      <span className="rounded-lg border-2 border-teal-600 bg-teal-100 px-3 py-2 text-xs font-semibold dark:border-teal-500 dark:bg-teal-950/50">2nd output neuron</span>
    </div>
    <p className="mb-0 mt-1 text-center text-xs text-slate-600 dark:text-slate-400">
      <em>Same weights</em> everywhere — that&apos;s <strong>weight sharing</strong>.
    </p>
  </div>
</div>

The **first output neuron** in the next layer is computed by applying this filter to the **top-left 3×3 patch** of the image.

The **next neuron** is computed by applying the **same filter** to the **next 3×3 patch**, shifted **one pixel** to the right.

This continues **across the entire image**, building up the **feature map** one position at a time. Step the kernel below to watch it happen — the same 9 weights are reused at every position:

## Why convolutions work so well

**Locality** and **weight sharing** are now familiar from [Lesson 4](./feature-detection-hierarchy). The third property is what makes convolutions especially powerful for images:

:::info[Translation invariance]

Because the **same filter** is applied at every spatial position, the network can **detect a pattern wherever it appears**. A cat-ear detector that fires in the top-left corner of one image will fire on a cat-ear in the bottom-right of another — no retraining needed.

:::

---

:::tip[Voila moments to remember]

- A **convolution** is a **filter sliding** across the image, performing **local pattern detection**.
- **Translation invariance**: the same filter applied at every position means a pattern is detected wherever it appears.

:::
