## Recurrence relations in RNNs

At the heart of an RNN lies the <strong>internal state</strong> <strong>h<sub>t</sub></strong>, which is updated as the network processes the input sequence <strong>step by step</strong>. The update to this state is defined by a <strong>recurrence relation</strong>, which incorporates both the current input <strong>x<sub>t</sub></strong> and the prior hidden state <strong>h<sub>t−1</sub></strong>. This recurrence relation enables the model to <strong>retain and evolve a memory</strong> of the sequence it has observed so far.

The state update can be formally described as:

$$
\mathbf{h}_t = \sigma\!\left( \mathbf{W}_{xh}\mathbf{x}_t + \mathbf{W}_{hh}\mathbf{h}_{t-1} + \mathbf{b}_h \right)
$$

$$
\mathbf{y}_t = \mathbf{W}_{hy}\mathbf{h}_t + \mathbf{b}_y
$$

## RNN logic through pseudocode

To further solidify this idea, consider a simple pseudocode example that illustrates how an RNN operates over a sequence:

```python
# Initialize RNN hidden state
h = 0
# Input sequence of words
sentence = ["I", "love", "recurrent", "neural"]
# RNN loop over time
for word in sentence:
    h = update_hidden_state(x=word, h_prev=h)
    y_hat = predict_output(h)
```

In this code snippet, each word in the input sequence is processed <strong>one at a time</strong>. The RNN uses the <strong>current word</strong> and the <strong>previously computed hidden state</strong> to compute a new hidden state. This new state is then used to produce an <strong>output prediction</strong>. The process is <em>iterative</em> and maintains a <strong>memory of the sequence</strong> through the hidden state updates.

## Defining loss over time

Training an RNN requires defining a <strong>loss function</strong> that quantifies the error between <strong>predicted</strong> and <strong>actual</strong> outputs. Unlike static models, where a <strong>single</strong> loss is computed, RNNs compute a loss <strong>at each time step</strong>. The <strong>total loss</strong> across a sequence is then the <strong>sum</strong> of the losses at individual steps:

$$
\mathcal{L}_{\text{total}} = \sum_{t=1}^{T} \mathcal{L}(\hat{\mathbf{y}}_t, \mathbf{y}_t)
$$

    This loss drives learning through <strong>Backpropagation Through Time (BPTT)</strong>, where gradients are computed not only over <em>network layers</em> but also <strong>across time steps</strong>.

## Implementation in code

Let us now translate this conceptual understanding into a more explicit <strong>code-level</strong> implementation.

Consider defining an RNN from scratch using an <strong>object-oriented</strong> approach:

```python
class SimpleRNN:
    def __init__(self, input_dim, hidden_dim, output_dim):
        self.W_xh = initialize_weights(input_dim, hidden_dim)
        self.W_hh = initialize_weights(hidden_dim, hidden_dim)
        self.W_hy = initialize_weights(hidden_dim, output_dim)
        self.b_h = initialize_bias(hidden_dim)
        self.b_y = initialize_bias(output_dim)
    def forward(self, input_sequence):
        h = np.zeros(self.hidden_dim)
        outputs = []
        for x_t in input_sequence:
            h = activation(np.dot(x_t, self.W_xh) + np.dot(h, self.W_hh) + self.b_h)
            y_hat = np.dot(h, self.W_hy) + self.b_y
            outputs.append(y_hat)
        return outputs
```

## From scratch to frameworks

Once you understand the fundamentals of how an RNN functions internally, you can transition to using <strong>high-level abstractions</strong> in frameworks such as <strong>TensorFlow</strong> and <strong>PyTorch</strong>. Both offer RNN modules and layers that encapsulate the forward pass, weight sharing, and even support for <em>batching</em> and <em>sequence padding</em>.

You will practice this in the <strong>upcoming lab</strong>, where you will implement RNNs for <strong>next-word prediction</strong>, <strong>character generation</strong>, and <strong>many-to-many sequence classification</strong> tasks.
