# Why attention? Moving beyond RNNs

Before **transformers**, **RNNs** and **LSTMs** were the main architectures for sequences. They read text **one token at a time**, like turning pages in order: each new word updates a **hidden state** that is supposed to compress everything seen so far.

That story is intuitive—but it collides with **modern hardware** and **long contexts**. The rest of this course replaces recurrence with **attention**; this lesson names **three concrete problems** so the design choices later feel inevitable.

## 1 · Sequential processing = no parallelization

Each position must be processed **after** the previous one—like **dominoes**: token 10 cannot finish until tokens 1–9 have updated the hidden state.

    <strong>Massive GPU parallelism</strong> vs <strong>an algorithm that is inherently serial along time</strong>.

## 2 · Long-range dependencies are hard

Information from an **early** token only reaches a **late** token by flowing through **every intermediate** hidden state. **LSTMs** (gates, cell state) **ease** this but **do not erase** it: over **50+** words, the signal is easy to dilute.

**Linguistic example** (subject–verb agreement across a long relative clause):

An RNN must carry that **agreement signal** through **every** step in the chain—there is no shortcut.

## 3 · Weak gradient flow (BPTT)

Training unrolls the network over the sequence and runs **backpropagation through time (BPTT)**. The gradient at an early timestep depends on a **product** of many Jacobian factors—one per hop.

<div className="not-prose my-6 rounded-xl border border-violet-200 bg-violet-50/60 p-4 font-mono text-xs leading-relaxed text-violet-950 dark:border-violet-900/40 dark:bg-violet-950/30 dark:text-violet-100 sm:text-sm">
  <p className="m-0 mb-2 text-center text-[10px] font-sans font-bold uppercase tracking-wider text-violet-800 dark:text-violet-300">Gradient path (conceptual)</p>
  <pre className="m-0 whitespace-pre-wrap text-center">{`∂L/∂h₁  ←  ···  ←  ∂L/∂h₁₀  ←  ∂L/∂h₂₀  ←  ···  ←  ∂L/∂h₁₀₀
         (each hop multiplies into the product)`}</pre>
</div>

<div className="not-prose my-6 grid gap-3 sm:grid-cols-2">
  <div className="rounded-xl border border-slate-300 bg-slate-100/90 p-4 dark:border-slate-600 dark:bg-slate-800/50">
    <p className="m-0 text-xs font-bold uppercase text-slate-600 dark:text-slate-400">Vanishing</p>
    <p className="mt-2 mb-0 text-sm text-slate-800 dark:text-slate-200">Factors shrink along the chain → early layers get **almost no update**.</p>
  </div>

  </div>
</div>

## Mental model: conveyor belt vs direct wires

<div className="not-prose my-6 overflow-x-auto rounded-xl border border-slate-200 dark:border-slate-700">
  <table className="w-full min-w-[300px] border-collapse text-left text-sm">
    <thead>
      <tr className="bg-slate-700 text-white dark:bg-slate-900">
        <th className="p-3 font-semibold">RNN / LSTM</th>
        <th className="p-3 font-semibold">Transformer (preview)</th>
      </tr>
    </thead>
    <tbody className="text-slate-800 dark:text-slate-200">
      <tr className="border-b border-slate-200 dark:border-slate-700">
        <td className="p-3">One <strong>conveyor belt</strong> of hidden states</td>
        <td className="p-3 bg-emerald-50/50 dark:bg-emerald-950/20">Many <strong>direct pairwise</strong> links per layer (attention weights)</td>
      </tr>
      <tr className="border-b border-slate-200 dark:border-slate-700">
        <td className="p-3">Distance from word <em>i</em> to <em>j</em> = <strong>O(|i − j|)</strong> steps along time</td>
        <td className="p-3 bg-emerald-50/50 dark:bg-emerald-950/20">Information can mix in <strong>one</strong> attention layer (subject to masks in decoders)</td>
      </tr>
      <tr>
        <td className="p-3">Whole-sequence summary squeezed into a <strong>fixed-size</strong> vector at each step</td>
        <td className="p-3 bg-emerald-50/50 dark:bg-emerald-950/20">Each position keeps a <strong>vector</strong>; attention <strong>selects</strong> what to read</td>
      </tr>
    </tbody>
  </table>
</div>

That shift—**parallelizable**, **direct paths**, **shorter gradient routes**—is why transformers scaled to today&apos;s language models. Next we implement the first ingredient: **dense token vectors** (embeddings).
