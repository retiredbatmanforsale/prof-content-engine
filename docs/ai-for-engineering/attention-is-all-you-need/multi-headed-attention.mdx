A **single** self-attention head computes **one** matrix of pairwise weights: for each position, **how much to read** from every other position. That is powerful—but **one** softmax competition can only shape **one** pattern at a time. **Multi-head attention** runs **h** attentions **in parallel** on **different learned projections**, then **merges** them so the model can hold **several relationship hypotheses** at once—syntax near the word, agreement far away, coreference, and more.

## Recall: one head (scaled dot-product self-attention)

Given projected matrices **Q, K, V** (same sequence length **n**, dimension **d_k** for **Q** and **K**, **d_v** for **V**):

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right) V
$$

The **softmax** is applied **row-wise**: for each query row **i**, you get **n** nonnegative weights summing to **1**—a **distribution** over positions. Stacking those rows gives the **attention matrix** **A** ∈ ℝⁿˣⁿ (conceptually; batching adds a batch dimension).

**Interpretation:** row **i** of **A** is a **heatmap** over keys: “from position **i**, how strongly am I looking at each **j**?”

---

## What changes with **multiple** heads?

**Idea:** instead of **one** triple **(Q, K, V)** in ℝⁿˣᵈ, use **h** **independent** linear maps from the **same** layer input **X** ∈ ℝⁿˣᵈᵐᵒᵈᵉˡ:

$$
Q_i = X W_i^Q,\quad K_i = X W_i^K,\quad V_i = X W_i^V
\quad\text{for } i = 1,\ldots,h
$$

Typically **d_k = d_v = d_model / h** so that concatenating heads restores width **h · d_v = d_model**.

Each head applies **Attention** with its **own** **Q_i, K_i, V_i**:

$$
\text{head}_i = \text{Attention}(Q_i, K_i, V_i)
$$

**Each head has its own attention matrix** **A_i** = row-softmax(**Q_i K_iᵀ / √d_k**): **h** different **n × n** **heatmaps** for the **same** sentence—different **match rules** (different **W_i^Q, W_i^K**), so different **patterns** light up.

### Merge heads (paper definition)

$$
\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1,\ldots,\text{head}_h)\, W^O
$$

Here **Q, K, V** in the paper’s boxed notation are often the **inputs from which** heads build their projections; in practice **MultiHead** takes the **layer input** **X** and learns all **W_i^Q, W_i^K, W_i^V** and **W^O**. The output has shape **(n, d_model)** after **W^O** ∈ ℝᵈᵐᵒᵈᵉˡˣᵈᵐᵒᵈᵉˡ.

(Exact wiring matches the **Transformer** figure in the original paper.)

## Why not one fat head? (The “debate”)

| | **Single head** (large **d_k**) | **Multi-head** (**h** heads, **d_k = d_model / h**) |
| --- | --- | --- |
| **Attention matrix** | One **n × n** map | **h** maps **A_1 … A_h**—**different subspaces** for matching |
| **Capacity** | One softmax “opinion” per query | **h** opinions, then **W^O** **mixes** channels |
| **Optimization** | Can be harder to train one entangled pattern | **Factorization** often **trains more reliably** (paper & practice) |
| **Interpretability** | One heatmap | **Several** heatmaps—easier to see **structure** in analysis |

**Intuition:** language **stack**s **patterns**—**words → phrases → clauses → discourse**. One head must **squeeze** all useful relations into **one** weight distribution per row; **multi-head** lets the network **allocate** different heads to **different** regularities, then **recombine** with **W^O**.

**Parameter count (conceptual):** you still spend parameters on **W_i^Q, W_i^K, W_i^V** and **W^O**; multi-head is not “free,” but it **restructures** the same **d_model** budget into **parallel** attention **experts**.

## Takeaway

- **Multi-head attention** = **h** scaled dot-product attentions in **parallel** on **learned projections**, **Concat**, then **W^O**.
- **Each head** has its **own attention matrix** (**heatmap**) over tokens.
- Compared to **one** self-attention head, multi-head gives **multiple simultaneous relationship views**, which aligns better with **layered structure** in language and **trains well** at scale.

## See every head light up — live

The math says each head learns its own pattern. You can actually see this. Below is GPT-2 (or DistilGPT-2 if you want a smaller download) running in your browser. Type a sentence, press **Run**, and you'll get a grid where each cell is one head's attention matrix at one layer.

What to look for:

- **Diagonal-heavy heads** — these mostly attend to the current token or its immediate neighbours. Often surface-level positional patterns.
- **"Previous-token" heads** — every row attends one column to the left. A genuine syntactic regularity the model discovers on its own.
- **Long-range heads** — bright spots far from the diagonal. Try `When Mary visited Paris, she ate a croissant.` and see which head connects `she` to `Mary` (coreference).
- **How depth changes the picture** — early layers tend to be local and crisp; deeper layers smear into broader, semantic patterns.

Click any cell to expand it into a full token-labeled view. Rows = which token is doing the attending; columns = which token it's attending to.
