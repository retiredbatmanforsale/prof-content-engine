# Residual connections

A **residual connection** ( **skip connection** ) wraps each transformer **sublayer**: the block’s output is **input plus whatever the sublayer learned to add**, not “replace input entirely.” That simple **+** is what lets **very deep** stacks of attention and feed-forward layers **train** without signals dying out—it gives gradients a **direct highway** and makes **optimization** far more **stable**.

## The core idea (identity + residual)

Let **x** be the input to a sublayer (tensor of shape **(batch, seq, d_model)**). A **residual block** computes:

$$
\text{output} = x + \text{Sublayer}(x)
$$

**Sublayer** is either **multi-head self-attention** (possibly masked), **cross-attention**, or the **feed-forward network**. The **Sublayer** is encouraged to learn a **correction** or **refinement** **Δ(x)** on top of **x**, instead of having to **re-encode** everything from scratch.

## 1 · Gradient flow

During **backpropagation**, the loss derivative with respect to **x** receives contributions from **both** paths:

- Through **Sublayer(x)** (the usual deep path through weights).
- Through the **skip** that adds **x** unchanged—an **identity** with derivative **1** with respect to **x**.

So even if the sublayer’s Jacobian is sometimes **small** or **ill-conditioned**, part of the gradient can still **bypass** that block. In **deep** transformers (many layers, each with **two** sublayers), that **repeated** “+1 highway” makes **vanishing** gradients **less severe** than in a plain stack of layers with **no** skip.

## 2 · Identity mapping

If **Sublayer** outputs **zero** (or is ignored early in training), **output = x**: the layer acts as the **identity**. The network does **not have to learn** “copy input” from scratch; it only learns **how much** to **deviate**. That matches the **ResNet** insight: **residuals** approximate **identity mappings** plus small **perturbations**, which is a **simpler** optimization landscape than learning arbitrary maps layer by layer.

---

## 3 · Easier optimization

Training is **easier** when:

- The model can **start** near **identity** and **grow** useful structure in **Δ**.
- **Attention** and **FFN** can focus on **what to change** rather than **rebuilding** the full representation every time.

Empirically, **transformer** training without residuals (or with broken skips) degrades badly—residuals are not optional decoration for deep stacks.

---

## 4 · Stability

**Layer normalization** (next lesson) is usually applied **around** the residual path (**Add &amp; Norm**). Together, **residual + norm** keeps activations in a **controlled range** across depth and makes **large** models **trainable**. Residuals address **gradient** path length; **norm** addresses **scale**—both matter for **stability**.

## Post-norm vs pre-norm

[Vaswani et al.](https://arxiv.org/abs/1706.03762) use **post-norm**: normalize **after** the residual add. Many later models use **pre-norm**: normalize **before** the sublayer, then add—often **more stable** when training **very deep** or **wide** models.

In **both** cases the **residual add** is what creates the **identity** path; **norm placement** changes **where** scaling happens in the block.

## Takeaway

| Theme | Role of residuals |
| ----- | ----------------- |
| **Gradient flow** | **Bypass** sublayers so signals survive **many** blocks |
| **Identity mapping** | **x + Δ** lets layers **refine** instead of **replace** |
| **Optimization** | **Simpler** learning dynamics in **deep** stacks |
| **Stability** | Works with **LayerNorm** to train **large** transformers |
