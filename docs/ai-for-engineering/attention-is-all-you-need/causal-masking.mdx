# Causal masking

Language models that predict **the next token** from **everything so far** must never be allowed to **peek** at the answer. **Self-attention** connects **all** pairs of positions in one layer—so without a **mask**, position **i** could attend to token **i + 1**, **i + 2**, … and the “prediction” would be **trivially** informed by the future. **Causal masking** enforces **autoregressive** structure: at each position you may only aggregate **past and current** tokens.

  <p className="m-0 text-sm text-slate-800 dark:text-slate-200">
    This is the **masked multi-head self-attention** block in the **decoder** of [Vaswani et al. (2017)](https://arxiv.org/abs/1706.03762). The same logic drives **decoder-only** models in the **GPT family** (and similar LMs): one **causal** attention pattern, repeated every layer. See also [Attention is All You Need — Lex AI](https://aiayn.aiseekhegaindia.com/) for the decoder flow.
  </p>
</div>

## Why do we need masking at all?

**Goal:** learn **P**(token at position **t** | tokens at positions **&lt; t**).

During **training**, the whole sequence is usually present on GPU—but we still must **simulate** generation: at time **t** the model must **not** see **t + 1, t + 2, …**. If it did:

- The task would **cheat**; loss would not match **inference** (where the future does not exist).
- There would be **no** well-defined **next-token** distribution conditioned only on the prefix.

**Causal masking** makes **teacher forcing** honest: you compute all positions in parallel for speed, but the **attention pattern** is **lower-triangular**, identical to what you would get if you unrolled **strictly left-to-right**.

## What is causal masking?

Before **softmax**, build attention **scores** **S = QKᵀ / √d_k**. For every pair **(i, j)** with **j &gt; i** (query at **i**, key at **j** — **future** key), set **S_ij** to **−∞** (or a very large negative number). Then **softmax** along the row assigns weight **0** to those positions.

So each row **i** of the attention matrix is a **probability distribution** only over **j ≤ i**: **past + present**, never **future**.

    <strong>Allowed</strong> region of the **n × n** attention matrix = <strong>lower triangle</strong> including the diagonal. <strong>Upper triangle</strong> (strict future) = <strong>masked out</strong>.

</div>

## Where causal masking is used (and where it is not)

**Encoder vs decoder (seq2seq):** the **encoder** “reads” the input sentence **all at once**—no future tokens to hide in the usual sense. The **decoder** “writes” the output **one token at a time**—that is where **causal** masking is **essential**. **GPT** drops the encoder entirely; **every** self-attention layer stays **causal**.

**BERT vs GPT:** **BERT**-style models do **not** use this **upper-triangle** causal attention mask on their main stack; they use **full** self-attention over the sequence (subject to **padding** / **special** masks for **[MASK]** and segments). **GPT** **always** uses causal masking—hence the famous **unidirectional** context window during generation.

---

## Implementation note

Add the mask to **logits** before **softmax**; unmasked rows still sum to **1**. In libraries this is often a boolean **(n, n)** mask or an additive **(1, 1, n, n)** bias for broadcast over batch and heads.

---

## Takeaway

- **Causal masking** = **hide future keys** from each query so **P(next | past)** is well-defined and **train = inference**.
- Required in the **decoder** of the original Transformer and in **decoder-only** **GPT**-family models.
- **Not** used in the standard **encoder** stack or **encoder-only** bidirectional models like **BERT**—there the point is **full context**, not **autoregressive** generation inside that stack.
