# The encoder stack

The **encoder** is where the **source** sentence (or any input sequence) is turned into a **stack of contextual vectors**—one row per token—that the **decoder** will later **read** through **cross-attention**. Every lesson so far was a **piece** of this machine; this page **wires them in the exact order** used in [Vaswani et al. (2017), *Attention Is All You Need*](https://arxiv.org/abs/1706.03762). The [Lex AI interactive diagram](https://aiayn.aiseekhegaindia.com/) follows the same stacking.

## Before layer 1: input to the encoder

1. **Token embeddings** + **positional encodings** → one vector per source position (**d_model**).
2. No **causal** mask: every token may attend to every other token (**bidirectional** encoder self-attention).

The result is the tensor that enters **encoder layer 1**.

---

## One encoder layer (exact paper rhythm)

The diagram in the paper labels each wrap-up **“Add &amp; Norm”**: **add** a **residual** connection, then apply **LayerNorm**. In **post-norm** form (what the paper’s picture encodes):

$$
\begin{aligned}
x_1 &= \text{LayerNorm}\bigl( x + \text{MultiHeadSelfAttention}(x) \bigr) \\
x_{\text{out}} &= \text{LayerNorm}\bigl( x_1 + \text{FFN}(x_1) \bigr)
\end{aligned}
$$

So **inside one layer**:

## Repeat **N** times: **× 6** in the base model

The **base** Transformer in the paper uses **N = 6** **identical** encoder layers (same pattern, **different learned weights** per layer). The output of layer **ℓ** feeds layer **ℓ + 1**.

## Output of the encoder

The final tensor has shape **(batch, source_length, d_model)**. For the full **encoder–decoder** Transformer, this sequence is projected to **K** and **V** for **each** **encoder–decoder (cross-)attention** sublayer in the decoder—so the **source** is **re-read** at every decoder depth with **updated** decoder queries.

**Next:** the **decoder stack**—where **masked** self-attention and **cross-attention** appear.
