# How neural networks make predictions: the **forward pass**

After **loss** (Lesson 4), the natural question is: **where** do the predictions **ŷ** come from? The **forward pass** is the answer: start from **input** **x**, apply each layer in order (**linear** → **activation**), and end with **outputs**. In libraries, the same computation is what runs inside **`model.predict(...)`** (or **`forward`** in training mode without dropout noise)—**no weight updates**, just **math** from **input** to **output**.

This lesson uses one tiny architecture so you can **read** every symbol:

- **Input layer:** **3** neurons → feature vector **(x₁, x₂, x₃)**
- **One hidden layer:** **4** neurons → after sigmoid **(a₁, a₂, a₃, a₄)**
- **Output layer:** **2** neurons → after sigmoid **(y₁, y₂)**

**Parameters:** **W₁**, **b₁** (input → hidden), **W₂**, **b₂** (hidden → output). Activations use **sigmoid** **σ** everywhere here so each value stays in **(0, 1)**—good for **probability-style** heads (you could swap **ReLU** in the hidden layer in practice; the **forward** **recipe** stays the same).

Row **j** of **W₁** mixes **x** into pre-activation **z₁ⱼ**; row **k** of **W₂** mixes **a** into **z₂ₖ**.

## Step 1 — Hidden layer (linear, then sigmoid)

**Pre-activations** (one number per hidden neuron):

$$
\mathbf{z}^{(1)} = W_1 \mathbf{x} + \mathbf{b}_1
$$

**Activations** (apply **σ** to **each** component):

$$
a_j = \sigma\bigl(z^{(1)}_j\bigr), \quad j = 1,2,3,4
$$

**Expanded** (nested in the inputs)—neuron **j** of the hidden layer:

$$
a_j = \sigma\Bigl( (W_1)_{j1}\, x_1 + (W_1)_{j2}\, x_2 + (W_1)_{j3}\, x_3 + (b_1)_j \Bigr)
$$

So **a₁, a₂, a₃, a₄** are **four** separate sigmoids of **four** weighted sums of **x₁, x₂, x₃**.

---

## Step 2 — Output layer (linear on **a**, then sigmoid)

$$
\mathbf{z}^{(2)} = W_2 \mathbf{a} + \mathbf{b}_2, \qquad
y_k = \sigma\bigl(z^{(2)}_k\bigr), \quad k = 1, 2
$$

**Fully nested** through **x** (substitute each **aⱼ**):

$$
y_1 = \sigma\Biggl(
\sum_{j=1}^{4} (W_2)_{1j}\,
\sigma\Bigl( \sum_{i=1}^{3} (W_1)_{ji}\, x_i + (b_1)_j \Bigr)
+ (b_2)_1
\Biggr)
$$

$$
y_2 = \sigma\Biggl(
\sum_{j=1}^{4} (W_2)_{2j}\,
\sigma\Bigl( \sum_{i=1}^{3} (W_1)_{ji}\, x_i + (b_1)_j \Bigr)
+ (b_2)_2
\Biggr)
$$

That is the **whole** forward map **(x₁, x₂, x₃) ↦ (y₁, y₂)** in one glance: **nested** **σ** of **affine** functions—**composition** is what makes the net **nonlinear**.

---

## What **σ** does here

$$
\sigma(z) = \frac{1}{1 + e^{-z}}
$$

So **y₁** and **y₂** each live in **(0, 1)**. If this head is **binary** or **multi-label** **probability**, you might pair them with **BCE** (Lesson 4); if you need **two** logits that **compete** (single class out of two), you would often use **one** linear layer + **softmax** instead—same **forward** **recipe**, different **last** nonlinearity.

---

## Forward pass = **`predict`** in code

| Phase | What runs |
| ----- | --------- |
| **Training** | Forward pass → **loss** → **backward** → **weight** update |
| **Inference** | Forward pass **only** → return **y** |

So **`model.predict(x)`** (or **`forward`** with **no_grad**) is **exactly** the chain above: **multiply**, **add**, **σ**, repeat. **No** gradients required to **use** a trained model—only to **learn** **W₁, b₁, W₂, b₂**.

---

## One-line story

**Input** → **W₁x + b₁** → **σ** → **a** → **W₂a + b₂** → **σ** → **y**. That is **prediction**.
