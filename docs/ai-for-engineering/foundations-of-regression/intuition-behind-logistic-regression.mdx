# Intuition behind logistic regression

Instead of directly predicting **y**, we predict the **probability that y belongs to class 1**.

We use the <strong>sigmoid function</strong> to ensure the output is always between **0** and **1**.

## Why a different optimisation function like MLE (maximum likelihood estimation) instead of SSE for logistic regression?

In <strong>linear regression</strong>, <strong>SSE</strong> (sum of squared errors) works well because we&apos;re directly predicting a <strong>continuous</strong> value and minimizing the squared distance between predictions and actual values.

In <strong>logistic regression</strong>, we&apos;re predicting <strong>probabilities</strong> between 0 and 1, and using <strong>SSE</strong> would create a <strong>non-convex</strong> optimization problem. This means:

## MLE (maximum likelihood estimation)

<strong>MLE</strong> instead <strong>maximizes the probability</strong> of observing our training data given our model parameters.

For logistic regression, working with the <strong>negative log-likelihood</strong> (what we minimize) gives a <strong>convex</strong> problem, meaning:

<div className="not-prose my-6 grid gap-3 md:grid-cols-2">
  <div className="rounded-lg border border-emerald-300 bg-emerald-50/80 p-3 text-sm dark:border-emerald-800 dark:bg-emerald-950/25">
    <strong className="text-emerald-900 dark:text-emerald-200">There&apos;s only one global minimum</strong> (standard convex formulation)
  </div>
  <div className="rounded-lg border border-emerald-300 bg-emerald-50/80 p-3 text-sm dark:border-emerald-800 dark:bg-emerald-950/25">
    <strong className="text-emerald-900 dark:text-emerald-200">Gradient descent</strong> will converge to the optimal solution <strong>regardless of initialization</strong>
  </div>
  <div className="rounded-lg border border-emerald-300 bg-emerald-50/80 p-3 text-sm md:col-span-2 dark:border-emerald-800 dark:bg-emerald-950/25">
    <strong className="text-emerald-900 dark:text-emerald-200">Efficient optimization</strong> methods apply (first-order and beyond)
  </div>
</div>

Schematically, with labels **yᵢ ∈ {0,1}** and predictions **ŷᵢ = σ(zᵢ)**:

$$
\mathcal{L}(\boldsymbol{\theta}) = - \sum_{i} \Bigl[ y_i \log \hat{y}_i + (1 - y_i)\log(1 - \hat{y}_i) \Bigr]
$$

This is the loss equivalent to <strong>maximum likelihood</strong> for Bernoulli outcomes; it is <strong>convex in θ</strong> for standard logistic regression.

---

## Why do we need the sigmoid function?

In linear regression, we model the target variable as:

$$
y = \theta_0 + \theta_1 x_1 + \theta_2 x_2 + \cdots + \theta_n x_n
$$

(Your note used coefficients **0, 1, 2, …, n** as subscripts — here **θ₀ … θₙ** are the same idea: intercept plus weights on each feature.)

However, in <strong>classification</strong> problems, **y** is either **0** or **1** (for binary classification).

If we use the linear regression equation <em>directly</em> as a prediction of &quot;probability,&quot; it could predict <strong>any real number</strong> (like **−10** or **25**), which doesn&apos;t make sense for probabilities.

### Solution: use sigmoid to squash the output

To ensure that our predictions are always between **0** and **1**, we apply the sigmoid to the <strong>linear score</strong> **z** (not to conflate **z** with the probability):

$$
\sigma(z) = \frac{1}{1 + e^{-z}}
$$

where **z** is the linear equation:

$$
z = mx + c \quad\text{(one feature)}
$$

or, in general,

$$
z = \theta_0 + \theta_1 x_1 + \theta_2 x_2 + \cdots + \theta_n x_n
$$

Now **σ(z)** (not **z** itself) outputs a value strictly between **0** and **1**, which we interpret as a probability:

$$
\mathbb{P}(y = 1 \mid X) = \sigma\bigl(\theta_0 + \theta_1 x_1 + \cdots + \theta_n x_n\bigr) = \frac{1}{1 + \exp\bigl(-(\theta_0 + \theta_1 x_1 + \cdots + \theta_n x_n)\bigr)}
$$

If this probability is <strong>close to 1</strong>, we classify as <strong>class 1</strong>.

If it is <strong>close to 0</strong>, we classify as <strong>class 0</strong>.
