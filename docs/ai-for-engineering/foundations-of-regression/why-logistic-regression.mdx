# Why do we need logistic regression?

<strong>Linear regression</strong> is great for <strong>continuous</strong> target variables (e.g., predicting house prices), but it <strong>fails</strong> when dealing with <strong>classification</strong> problems (e.g., predicting if an email is spam or not).

      <strong>Discrete</strong> labels — spam / not spam — and usually <strong>probabilities</strong> over classes, not unbounded scores.

## Problems with linear regression for classification

### Output is unbounded

Linear regression predicts <strong>any real number</strong>, but we need <strong>probabilities</strong> (values between 0 and 1) for classification.

<div className="not-prose my-5 rounded-xl border-l-4 border-rose-500 bg-rose-50/80 py-3 pl-4 pr-3 dark:bg-rose-950/20">
  <p className="m-0 text-sm font-semibold text-rose-900 dark:text-rose-200">The mismatch</p>
  <p className="mt-2 mb-0 text-sm text-slate-800 dark:text-slate-200">
    A line <strong>ŷ = wᵀx + b</strong> can output <em>5</em>, <em>−0.3</em>, or <em>π</em>. That is fine for &quot;dollars&quot; but awkward for &quot;probability of class 1.&quot;
  </p>
</div>

<strong>Solution:</strong> We need a mechanism to map those real numbers into <strong>probabilities</strong> — for example, an output that always says the probability of a data point belonging to <strong>class 0</strong> is <strong>0.4</strong> and the probability of belonging to <strong>class 1</strong> is <strong>0.6</strong>.

### Non-linearity

First we need a way to represent <strong>non-linear</strong> structure in the world, and plain linear models like linear regression can&apos;t really capture that on their own. Second, we often want <strong>decision boundaries</strong> that separate regions of the input space by class — and treating classification as raw linear regression on labels doesn&apos;t give us a clean probabilistic separator.

## From linear score to probability: the sigmoid bridge

To fix these problems, we <strong>squash</strong> the output of a linear model into a probability using the <strong>sigmoid function</strong>, which leads us to <strong>logistic regression</strong>.

Let <strong>z = wᵀx + b</strong> be the same linear <em>score</em> you would use in linear regression (for one sample). The sigmoid is:

$$
\sigma(z) = \frac{1}{1 + e^{-z}}
$$

Key facts (easy to check from the formula):

$$
\sigma(z) \in (0,\,1), \qquad
\lim_{z \to -\infty} \sigma(z) = 0, \qquad
\lim_{z \to +\infty} \sigma(z) = 1.
$$

For binary classification, a standard choice is to interpret:

$$
\mathbb{P}(y = 1 \mid \mathbf{x}) = \sigma\bigl(\mathbf{w}^\top \mathbf{x} + b\bigr).
$$

Then (for two classes):

$$
\mathbb{P}(y = 0 \mid \mathbf{x}) = 1 - \sigma\bigl(\mathbf{w}^\top \mathbf{x} + b\bigr).
$$

The set of points where <strong>σ(wᵀx + b) = 0.5</strong> (equivalently <strong>wᵀx + b = 0</strong>) is still a <strong>linear decision boundary</strong> in **x** — logistic regression is <em>linear in the features</em> but <em>non-linear in the probability</em> because of **σ**. Richer boundaries usually mean richer features (e.g. polynomials) or a different model — but we still start from this clean <strong>score → probability</strong> idea.
