# Log likelihood instead of squared error

In **linear regression** we often minimize the **sum of squared errors (SSE)**:

$$
J_{\text{SSE}} = \sum_{i=1}^{m} \bigl( y_i - \hat{y}_i \bigr)^2
$$

where **ŷᵢ** is a **real-valued** prediction (same scale as **yᵢ**).

For **logistic regression**, **ŷᵢ = σ(zᵢ)** is a **probability** in **(0, 1)**. If we plug that into **squared error** on **yᵢ ∈ {0, 1}**, the resulting loss as a function of the parameters is **not convex** in general: **gradient descent** can get stuck in **local minima** and you **lose a guarantee** of reaching a global optimum.

**Maximum likelihood estimation (MLE)** picks parameters that make the observed labels **most probable** under the model. For standard logistic regression, turning that into a minimization problem (**negative log-likelihood**) gives a **convex** objective—much nicer for gradient-based optimization.

## Step 1 — Bernoulli probabilities for one example

For binary labels **yᵢ ∈ {0, 1}**, let the model output **ŷᵢ = ℙ(yᵢ = 1 | xᵢ)** (your **sigmoid** of the linear score). Then:

$$
\mathbb{P}(y_i = 1 \mid x_i) = \hat{y}_i, \qquad
\mathbb{P}(y_i = 0 \mid x_i) = 1 - \hat{y}_i
$$

We can write the **probability of the observed label** **yᵢ** in one expression:

$$
\mathbb{P}(y_i \mid x_i) = \hat{y}_i^{\,y_i}\,(1 - \hat{y}_i)^{\,1 - y_i}
$$

- If **yᵢ = 1**, this reduces to **ŷᵢ**.
- If **yᵢ = 0**, it reduces to **1 − ŷᵢ**.

  <p className="m-0 text-sm text-slate-800 dark:text-slate-200">
    **ŷᵢ** is the **predicted** probability (depends on **θ**). Do not confuse it with the **label** **yᵢ**.
  </p>
</div>

## Step 2 — Likelihood for the whole dataset

Assume examples are **independent** (standard i.i.d. story). The **likelihood** is the **product** over all **m** points:

$$
L(\boldsymbol{\theta}) = \prod_{i=1}^{m} \hat{y}_i^{\,y_i}\,(1 - \hat{y}_i)^{\,1 - y_i}
$$

**MLE** chooses **θ** to **maximize** **L(θ)** (equivalently **log L**).

---

## Step 3 — Log-likelihood (turn the product into a sum)

$$
\log L(\boldsymbol{\theta}) = \sum_{i=1}^{m} \Bigl[ y_i \log \hat{y}_i + (1 - y_i)\log(1 - \hat{y}_i) \Bigr]
$$

Products of many numbers in **(0, 1)** underflow quickly; the **log** makes the objective a **sum**—stable and easy to differentiate.

We **maximize** **log L**, so in code we usually **minimize** the **negative**. Averaging over **m** gives a **mean** loss:

$$
J(\boldsymbol{\theta}) = - \frac{1}{m} \sum_{i=1}^{m} \Bigl[ y_i \log \hat{y}_i + (1 - y_i)\log(1 - \hat{y}_i) \Bigr]
$$

This **J** is the **binary cross-entropy** (also called **log loss**): the standard **cost function** for logistic regression under MLE.

## Sanity check with two probabilities

| **yᵢ** | Term inside the sum |
| -----: | ------------------- |
| **1**  | **log ŷᵢ** (want **ŷᵢ** large) |
| **0**  | **log(1 − ŷᵢ)** (want **ŷᵢ** small) |

If the model is **confident and wrong** (e.g. **yᵢ = 1** but **ŷᵢ ≈ 0**), **log ŷᵢ** is very negative and **J** spikes—exactly the penalty we want.
