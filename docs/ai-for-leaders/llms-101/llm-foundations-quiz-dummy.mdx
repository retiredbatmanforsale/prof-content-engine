    <ol className="m-0 space-y-2 text-sm text-slate-800 dark:text-slate-200">
      <li>
        <strong>1.</strong> B — LLMs are trained to produce fluent, context-fitting continuations; factual accuracy is not guaranteed.
      </li>
      <li>
        <strong>2.</strong> B — Autoregressive = one token at a time, conditioned on everything generated so far.
      </li>
      <li>
        <strong>3.</strong> C — ChatGPT-style assistants are built around a <strong>decoder-only</strong> transformer.
      </li>
      <li>
        <strong>4.</strong> C — Tokenization splits text into model-specific units and maps them to vocabulary IDs.
      </li>
      <li>
        <strong>5.</strong> C — Embeddings are dense vectors capturing meaning and usage; IDs and position are separate mechanisms.
      </li>
      <li>
        <strong>6.</strong> B — Softmax turns logits over the vocabulary into a probability distribution for the next token.
      </li>
      <li>
        <strong>7.</strong> C — Inference = forward passes through a fixed (trained) model to produce outputs for users.
      </li>
      <li>
        <strong>8.</strong> B — Models optimize for likelihood of training text, not for verified truth; hallucinations are a known issue.
      </li>
      <li>
        <strong>9.</strong> B — Conversational behavior comes largely from fine-tuning (and related alignment) on dialogue-style data.
      </li>
      <li>
        <strong>10.</strong> B — Standard supervised training: forward → loss → backward (gradient) → weight update, repeated.
      </li>
    </ol>
  </div>
</details>
