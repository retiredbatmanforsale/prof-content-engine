  solution={`
import numpy as np

def layer_norm(x, gamma, beta, eps=1e-5):
    mu = x.mean(axis=-1, keepdims=True)
    var = x.var(axis=-1, keepdims=True)
    x_norm = (x - mu) / np.sqrt(var + eps)
    return gamma * x_norm + beta
`}
  hints={[
    "Compute mean and variance along the LAST axis (keepdims=True so they broadcast back). Then z-score: (x - mu) / sqrt(var + eps). Then gamma * x_norm + beta.",
    "Important: axis=-1 only. LayerNorm normalises per-token (each row of the C-dim feature axis is its own normalisation universe). NOT per batch — that's BatchNorm, a different thing.",
    "The eps inside the sqrt prevents division by zero when the variance is exactly 0 (constant input). Don't add eps OUTSIDE the sqrt — that changes the math when var > 0.",
  ]}
>

LayerNorm — the per-token normalisation that re-establishes unit variance at the start of every sub-layer in a transformer. Without it, activations across stacked blocks grow unboundedly; the residual stream's magnitudes blow up, attention saturates, training diverges.

The math:

`x_norm = (x - mean(x)) / sqrt(var(x) + eps)`

`out = gamma * x_norm + beta`

Where `mean` and `var` are computed **along the last axis only** (per token, not per batch). `gamma` and `beta` are learnable parameters of shape `(C,)` — `gamma` initialised to 1, `beta` initialised to 0 (so LN starts as the identity).

The diagnostic the harness checks: with `gamma=1, beta=0`, each token's output should have mean ≈ 0 and std ≈ 1 along the feature axis, regardless of input scale. If you accidentally normalise across the batch axis (BatchNorm), the test fails immediately on the first `(B, T, C)` input — those statistics would mix tokens across positions and across batch elements.
