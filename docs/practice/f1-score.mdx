import numpy as np

def f1_score(y_pred, y_true):
    tp = int(np.sum((y_pred == 1) & (y_true == 1)))
    fp = int(np.sum((y_pred == 1) & (y_true == 0)))
    fn = int(np.sum((y_pred == 0) & (y_true == 1)))
    if tp + fp == 0 or tp + fn == 0:
        return 0.0
    precision = tp / (tp + fp)
    recall = tp / (tp + fn)
    if precision + recall == 0:
        return 0.0
    return 2 * precision * recall / (precision + recall)
`}

F1 score — the harmonic mean of precision and recall, the standard one-number summary of a binary classifier.

The math:

`precision = TP / (TP + FP)`

`recall = TP / (TP + FN)`

`F1 = 2 · precision · recall / (precision + recall)`

Why harmonic mean rather than arithmetic? Because it punishes imbalance — a model with `precision=1.0, recall=0.01` gets `F1 ≈ 0.02`, not `0.5`. Arithmetic mean would let one strong half mask the other.

Three edge cases the harness checks:

1. **No positive predictions** (`TP + FP = 0`) → return `0.0`. Don't let `0/0` produce NaN.
2. **No positive labels in the test set** (`TP + FN = 0`) → return `0.0`. Same reason.
3. **Symmetry between precision and recall**: a `(P=1.0, R=0.2)` model and a `(P=0.2, R=1.0)` model should have identical F1. The formula doesn't distinguish "precise but timid" from "noisy but exhaustive" — that's by design.

For multi-class, you'd compute F1 per class and average (macro / weighted / micro). For now: binary only.
