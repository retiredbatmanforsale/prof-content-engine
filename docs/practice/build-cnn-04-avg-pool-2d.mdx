import numpy as np

def avg_pool_2d(x, kernel_size, stride):
    C, H, W = x.shape
    k, s = kernel_size, stride
    out_H = (H - k) // s + 1
    out_W = (W - k) // s + 1
    out = np.zeros((C, out_H, out_W))
    for c in range(C):
        for i in range(out_H):
            for j in range(out_W):
                out[c, i, j] = x[c, i*s:i*s+k, j*s:j*s+k].mean()
    return out
`}

Average pooling — the gentler twin of max pooling. Same loop structure, same shape formula, but reduces a window with `mean` instead of `max`.

Why both pooling ops? They have different inductive biases:

- **Max pool** says "the strongest signal in this region wins" — sparse gradients, translation-invariance, sharp feature detection.
- **Avg pool** says "smooth the signal across this region" — dense gradients (every input contributes equally), more like a fixed low-pass filter.

Modern architectures use **average pool** primarily as a *global* pool at the end of the network — turning a `(C, H, W)` feature map into a `(C,)` vector before the final linear classifier. ResNet, MobileNet, and Vision Transformers all do this. **Max pool** is more common between conv blocks for spatial downsampling, though `stride=2` convolutions have largely replaced both in modern CNNs.

The implementation is the same three nested loops as `build-cnn-03`, with `.max()` swapped for `.mean()`. The diagnostic test the harness probes for: divide by `k*k`, not by anything else.
