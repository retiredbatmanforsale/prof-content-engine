import numpy as np

def attention(Q, K, V, mask=None):
    d = Q.shape[-1]
    scores = Q @ K.T / np.sqrt(d)
    if mask is not None:
        scores = np.where(mask, scores, -1e9)
    # numerically-stable softmax along the last axis
    scores = scores - scores.max(axis=-1, keepdims=True)
    weights = np.exp(scores)
    weights = weights / weights.sum(axis=-1, keepdims=True)
    return weights @ V
`}

The single most-asked operation in modern deep learning. Every attention head — self, cross, multi-head, grouped-query — bottoms out in this function. Implement the single-head, self-attention version.

The formula:

`Attention(Q, K, V) = softmax(Q @ K^T / sqrt(d_k)) @ V`

Three subtleties the harness probes for:

1. **The sqrt(d_k) scale.** Without it, dot-product magnitudes grow with the embedding dimension. At `d_k = 64` (GPT-2's per-head size), softmax saturates to a one-hot and gradients vanish.
2. **Mask placement.** The mask is applied to **scores**, not weights — set blocked positions to a large negative number *before* softmax so they go to ~0 probability *after*. Zeroing weights post-softmax leaves the surviving weights summing to less than 1.
3. **Numerical-stable softmax.** Same `scores - scores.max(axis=-1)` trick as the standalone softmax problem.

This is a series problem — your function name `attention(Q, K, V, mask=None)` is the contract that `build-gpt-06-multi-head-attention` will call later.
