import numpy as np

def apply_repetition_penalty(logits, prefix_ids, penalty):
    # Apply the asymmetric rule once per UNIQUE token id (set deduplicates).
    for t in set(int(i) for i in prefix_ids):
        if logits[t] > 0:
            logits[t] /= penalty
        elif logits[t] < 0:
            logits[t] *= penalty
        # logits[t] == 0: leave it.
    return logits
`}

The HuggingFace-style **repetition penalty** — the standard fix when an LLM is looping ("the the the the…") and you don't want to fully shut off temperature-driven sampling. Down-weight the logits of already-emitted tokens so they're less likely to be picked again.

The asymmetric rule (Keskar et al. 2019, CTRL):

`if logits[t] > 0:   logits[t] /= penalty`

`if logits[t] < 0:   logits[t] *= penalty`

Why asymmetric? Because we want the penalty to **push the logit toward zero or below**, regardless of its current sign. If you naively `divide` a negative logit, you'd make it *less negative* — i.e., more likely. The sign-aware rule fixes this; both branches push the logit downward.

Three subtleties the harness checks:

1. **Sign-aware update** — both directions of the asymmetric rule.
2. **Penalty applied once per unique token** — even if the prefix contains the same id 5 times, the logit changes by one factor of `penalty`, not five.
3. **In-place mutation** — same reference returned.

Typical setting: `penalty = 1.1` to `1.3`. Higher values produce too-aggressive de-repetition; lower values barely move the distribution.
