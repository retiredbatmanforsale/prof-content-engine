  solution={`
import numpy as np

def greedy_decode(predict_fn, prefix, max_new_tokens):
    ids = list(prefix)  # copy so we don't mutate the caller's list
    for _ in range(max_new_tokens):
        logits = predict_fn(ids)
        next_id = int(np.argmax(logits))
        ids.append(next_id)
    return ids

Greedy decoding is the simplest autoregressive generation strategy and the baseline every other decoding rule is measured against. Implement the loop.

The pattern is universal across every autoregressive language model:

`while not done:`
&nbsp;&nbsp;&nbsp;&nbsp;`logits = model(current_ids)`
&nbsp;&nbsp;&nbsp;&nbsp;`next_id = argmax(logits)`
&nbsp;&nbsp;&nbsp;&nbsp;`current_ids.append(next_id)`

The pathologies of greedy decoding — repetition cycles, no diversity, dead air on creative tasks — are exactly why temperature, top-k, and top-p exist. But greedy is the right *default* when you want determinism: classifier outputs, code completion, structured-output extraction.

Two subtleties the harness probes for:

1. **The prefix is preserved unchanged in the output.** `out[:len(prefix)] == list(prefix)`. If you mutate the input list, you've broken the contract.
2. **`predict_fn` is called exactly `max_new_tokens` times**, with the prefix growing by one each call. The "transition table" diagnostic test wires `predict_fn` to follow a 4-cycle, and the output should follow that cycle deterministically.

Cast the argmax to a python `int` before appending — numpy ints can confuse downstream code that expects pure-python types.
