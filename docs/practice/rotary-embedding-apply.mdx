import numpy as np

def apply_rotary(x):
    B, T, C = x.shape
    assert C % 2 == 0, "C must be even"
    # Pair up the last axis: (B, T, C/2, 2)
    x_pairs = x.reshape(B, T, C // 2, 2)
    x_e = x_pairs[..., 0]   # even-indexed dims
    x_o = x_pairs[..., 1]   # odd-indexed dims

    # Per-(t, i) rotation angle.
    pos = np.arange(T)[:, None]                       # (T, 1)
    i = np.arange(0, C, 2)                            # (C/2,)
    inv_freq = 1.0 / (10000 ** (i / C))               # (C/2,)
    theta = pos * inv_freq                            # (T, C/2)
    cos_t = np.cos(theta); sin_t = np.sin(theta)      # both (T, C/2)

    y_e = x_e * cos_t - x_o * sin_t
    y_o = x_e * sin_t + x_o * cos_t
    return np.stack([y_e, y_o], axis=-1).reshape(B, T, C)
`}

Rotary Position Embedding — [Su et al. (2021)](https://arxiv.org/abs/2104.09864). The positional encoding scheme that displaced sinusoidal and learned positional embeddings in modern LLMs (Llama, Mistral, Qwen, Gemma all use it). Applied directly to query and key vectors before attention, not added to the embeddings.

The mechanic: split each token's `(C,)` feature vector into `C/2` pairs, then rotate each pair `(x_{2i}, x_{2i+1})` by an angle `θ = t / 10000^(2i/C)`:

`new = [cos · x_e − sin · x_o,   sin · x_e + cos · x_o]`

Three properties that make it work for attention:

1. **Norm-preserving.** A rotation in 2D preserves vector length, so RoPE doesn't change `||q||` or `||k||` — it only re-aligns their angles. Attention scores `q · k` then capture both content similarity AND relative position via the angle difference.
2. **Position 0 is the identity.** `t=0 → θ=0 → cos(0)=1, sin(0)=0` → no rotation. The first token is unchanged.
3. **Frequency spread.** Different pairs rotate at different rates: low-index pairs rotate fast (`θ ∝ t`), high-index pairs slow (`θ ∝ t / 10000`). Attention can pick out short-range or long-range patterns by attending to different frequency bands.

The diagnostic the harness checks: norm preservation across every position. Get the cos/sin signs wrong and rotations turn into reflections — norms still equal, but the test for position-0-identity fails.
