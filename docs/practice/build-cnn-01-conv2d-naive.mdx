import numpy as np

def conv2d_naive(x, W):
    C_in, H, Wd = x.shape
    C_out, _, kH, kW = W.shape
    out_H = H - kH + 1
    out_W = Wd - kW + 1
    out = np.zeros((C_out, out_H, out_W))
    for o in range(C_out):
        for i in range(out_H):
            for j in range(out_W):
                # Element-wise product over the receptive field, summed over (C_in, kH, kW).
                out[o, i, j] = np.sum(x[:, i:i+kH, j:j+kW] * W[o])
    return out
`}

The first piece of every CNN — and the most-asked CV implementation question. Implement 2D convolution **forward** for a single example, with stride 1 and no padding. We use the deep-learning convention (cross-correlation, no kernel flip).

The shapes:

`x: (C_in, H, W)` — input feature map

`W: (C_out, C_in, kH, kW)` — kernel

`out: (C_out, H - kH + 1, W - kW + 1)`

For each output `(c_out, i, j)`:

`out[c_out, i, j] = sum over c_in, kh, kw of: x[c_in, i+kh, j+kw] * W[c_out, c_in, kh, kw]`

The naive implementation is three nested loops (over `c_out`, `i`, `j`), with the inner reduction handled by `np.sum` on a sliced patch. It's slow — O(C_out · H · W · C_in · kH · kW) — but it's the right starting point. Faster approaches (im2col, FFT, the Toeplitz trick) build on the same indexing.

In `build-cnn-02` you'll add stride and padding. In `build-cnn-05` you'll write the backward pass.
