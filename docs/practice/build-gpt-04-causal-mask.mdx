  solution={`
import numpy as np

def build_causal_mask(T):
    return np.tril(np.ones((T, T), dtype=bool))


def apply_causal_mask(scores):
    T = scores.shape[-1]
    mask = build_causal_mask(T)
    return np.where(mask, scores, -np.inf)
`}
  hints={[

    "Using -1e9 instead of -np.inf also works in practice (and is friendlier to fp16). Both pass the harness, but -inf is conceptually cleaner.",

>

GPT-2 is a *decoder* — at training time, predicting token `t+1` from tokens `0..t` requires position `t` cannot attend to positions `> t`. Otherwise the model "cheats" by reading the answer directly from its input. The fix is one matrix and one `where` call.

Two functions:

1. `build_causal_mask(T)` — a `(T, T)` bool mask. `True` at and below the diagonal, `False` above.
2. `apply_causal_mask(scores)` — given a `(T, T)` score matrix from `Q @ K.T / sqrt(d)`, replace the upper triangle with `-inf` so softmax kills it.

The diagnostic test is the one that matters: after the mask is applied and softmax is taken, positions `j > i` must have **exactly** 0 probability — not 0.001, not 0.0001. That's only true if you mask **before** softmax with `-inf` (or a very large negative number). Zeroing weights *after* softmax leaves the rest of the row summing to less than 1, and changes the gradient incorrectly.
