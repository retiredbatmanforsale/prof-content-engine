import numpy as np

def temperature_scale(logits, T):
    z = logits / T
    z = z - z.max()           # numerical stability — same trick as plain softmax
    e = np.exp(z)
    return e / e.sum()

Temperature is the simplest knob in next-token sampling. One scalar `T > 0` reshapes the entire output distribution between "fully deterministic" and "pure noise" without changing the model.

The math:

`p'_i = exp(z_i / T) / sum_j exp(z_j / T)`

Three regimes:

- **`T → 0+`** sharpens to a one-hot at the argmax — equivalent to greedy decoding.
- **`T = 1`** is plain softmax — no change to the distribution.
- **`T → ∞`** flattens toward uniform — all tokens equally likely.

Common settings: `T ∈ [0.7, 1.0]` for natural-sounding LLM output. Higher gets weird and creative; lower gets stiff and repetitive. The OpenAI Playground default is 1.0 with optional top-p; ChatGPT typically uses lower (~0.7).

The single subtle point: **numerical stability**. The naive `exp(z/T)` overflows for large logits — exactly the same way as the original softmax problem. Subtract the max after the temperature division, before the exp. The harness probes this with `[1000, 1001, 1002]`.
