import numpy as np

def _ref(x, W, padding=0, stride=1):
    C_in, H, Wd = x.shape
    C_out, _, kH, kW = W.shape
    if padding > 0:
        x = np.pad(x, ((0, 0), (padding, padding), (padding, padding)))
    H_pad = H + 2 * padding; W_pad = Wd + 2 * padding
    out_H = (H_pad - kH) // stride + 1
    out_W = (W_pad - kW) // stride + 1
    out = np.zeros((C_out, out_H, out_W))
    for o in range(C_out):
        for i in range(out_H):
            for j in range(out_W):
                hi = i * stride; wi = j * stride
                out[o, i, j] = np.sum(x[:, hi:hi+kH, wi:wi+kW] * W[o])
    return out

    C_in, H, Wd = x.shape
    C_out, _, kH, kW = W.shape
    if padding > 0:
        x = np.pad(x, ((0, 0), (padding, padding), (padding, padding)))
    H_pad = H + 2 * padding
    W_pad = Wd + 2 * padding

The conv2d that real CNNs actually use — with zero-padding and stride. Two extra parameters on top of `build-cnn-01`, and the output shape formula expands to:

`out_H = (H + 2*padding - kH) // stride + 1`

Two patterns the harness probes for:

1. **"Same" padding.** With `padding = (k - 1) // 2` and `stride = 1`, the spatial dims are preserved. Every CNN architecture uses this somewhere — it lets you stack conv layers without shrinking the feature map. The diagnostic test checks it for `k = 3, padding = 1`.
2. **Padding mode.** Always **zero**, never edge-replicate or reflect. `np.pad(x, ((0,0), (p,p), (p,p)))` defaults to zeros, which is what you want.

The padding-and-stride conv is the bedrock of CNN architectures. After this, `build-cnn-03/04` add pooling, `build-cnn-05` adds the backward pass, and `build-cnn-06` composes the full forward of a tiny image classifier.
