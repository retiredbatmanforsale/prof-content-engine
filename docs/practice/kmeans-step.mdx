import numpy as np

def kmeans_step(X, centroids):
    # 1. Distance from every point to every centroid: (N, K).
    diffs = X[:, None, :] - centroids[None, :, :]      # (N, K, D)
    dists = (diffs ** 2).sum(axis=-1)                  # (N, K) — squared L2
    # 2. Argmin gives the assignment.
    assign = dists.argmin(axis=1)                      # (N,)
    # 3. Update each centroid.
    new_centroids = centroids.copy()
    for k in range(centroids.shape[0]):
        mask = assign == k
        if mask.any():
            new_centroids[k] = X[mask].mean(axis=0)
        # else: leave unchanged
    return new_centroids, assign
`}

K-means — Lloyd's algorithm in two halves. Repeat until convergence:

1. **Assign** each data point to the nearest centroid (Euclidean distance).
2. **Update** each centroid to the mean of its assigned points.

Implement one iteration. The function returns `(new_centroids, assignments)` so a caller can compose it into a loop and track convergence.

Three details the harness probes:

1. **Empty clusters.** If a centroid lands somewhere no points are nearest to, its assigned set is empty and the mean is NaN. The standard convention (and what we test): **leave the empty centroid unchanged**. Production k-means often re-seeds it from a random point, but that introduces non-determinism we don't want here.
2. **Tie-breaking.** When two centroids are exactly equidistant from a point, `np.argmin` returns the lower-index centroid. Deterministic.
3. **Loss decreases.** Lloyd's algorithm guarantees within-cluster sum-of-squares is non-increasing per iteration. The harness runs 5 iterations and checks loss didn't increase — a sanity check that your update is moving centroids toward their cluster means, not away.

Used in vector quantisation, k-NN preprocessing, codebook learning for image compression, and as the initialisation strategy for many richer clustering algorithms.
