import numpy as np

def sinusoidal_pe(T, C):
    pe = np.zeros((T, C))
    pos = np.arange(T)[:, None]                       # (T, 1)
    i = np.arange(0, C, 2)                            # (C/2,) — even dims
    div = 10000 ** (i / C)                            # (C/2,) — frequency divisor
    angles = pos / div                                # (T, C/2)
    pe[:, 0::2] = np.sin(angles)                      # even columns
    pe[:, 1::2] = np.cos(angles)                      # odd columns
    return pe
`}

The original Transformer paper's positional encoding — fixed, parameter-free, and extrapolates to sequence lengths longer than anything in training. GPT-2 chose learned positional embeddings instead, but sinusoidal is the encoding every interview asks about because the math is non-trivial.

The formula:

`PE[t, 2i]     = sin(t / 10000^(2i/C))`

`PE[t, 2i+1]   = cos(t / 10000^(2i/C))`

Two properties that make it work:

1. **At position 0, all sin dims are 0 and all cos dims are 1** — a clean baseline. The harness checks this directly.
2. **No collisions across positions** — different rows of the table are different vectors. Without this, attention can't tell positions apart.

The deeper magic that the paper actually relied on: the encoding for position `t + k` is a linear function of the encoding for position `t`. That means attention can learn a "shift by k" operation as a fixed linear transform — the architecture's prior on relative position. We don't test that property explicitly here, but it's why this encoding was chosen over alternatives.
