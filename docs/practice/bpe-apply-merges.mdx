  solution={`
def apply_merges(tokens, merges):
    # Index merges by their pair → priority (lower = higher priority).
    rank = {pair: i for i, pair in enumerate(merges)}
    tokens = list(tokens)

    while True:
        # Find the highest-priority adjacent pair anywhere in tokens.
        best_idx, best_rank = None, None
        for i in range(len(tokens) - 1):
            r = rank.get((tokens[i], tokens[i + 1]))
            if r is not None and (best_rank is None or r < best_rank):
                best_rank = r
                best_idx = i
        if best_idx is None:
            return tokens
        # Merge the pair at best_idx.
        tokens = (
            tokens[:best_idx]
            + [tokens[best_idx] + tokens[best_idx + 1]]
            + tokens[best_idx + 2:]
        )
`}

Byte-pair encoding (BPE) trains a vocabulary by iteratively merging the most frequent adjacent token pair. At inference time, you apply those merges in priority order to a string of single-character tokens, and out comes the tokenization GPT-2 (or any BPE-based model) actually uses.

You're implementing **only the inference half** — the merge application. Training (counting pairs and picking the merge to add to the vocab) is a separate problem.

The algorithm is small but subtle:

1. You're given a list of single-character (or single-byte) tokens and a list of merges in priority order.
2. Find every adjacent pair in the token list. If any of them appears in the merges dict, pick the one with the **highest priority** (= lowest index in the merges list).
3. Merge that pair into a single token. Repeat from step 2.
4. When no adjacent pair appears in merges, you're done.

The tricky part is step 2: don't process merges one-by-one in list order. Each iteration scans the *current* tokens and picks the globally best pair. Otherwise you'll get the wrong tokenization for the test case where one merge enables another.
