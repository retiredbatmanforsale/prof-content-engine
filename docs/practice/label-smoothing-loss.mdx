import numpy as np

def label_smoothing_loss(logits, target, alpha):
    K = len(logits)
    # numerically-stable log_softmax
    z = logits - logits.max()
    log_probs = z - np.log(np.exp(z).sum())
    # Smoothed target: (1 - alpha) at the true class + alpha / K everywhere.
    # Loss = -sum(smoothed * log_probs)
    return float(-((1 - alpha) * log_probs[target] + (alpha / K) * log_probs.sum()))
`}

Label smoothing — replace the one-hot training target with a slightly softer distribution to discourage the model from becoming over-confident. Standard regularisation in vision (Inception-v3, ResNet) and machine translation (Transformer paper).

The smoothed target (PyTorch convention):

`soft[i] = (1 − α) · 1[i = y] + α / K`

So the true class gets `(1 − α + α/K)` and every other class gets `α/K`. Total mass still sums to 1.

The loss is the cross-entropy of `soft` against `softmax(logits)`:

`L = −Σ_i soft[i] · log_softmax(logits)[i]`

Why use it? Without smoothing, the model is rewarded for assigning probability 1 to the correct class — which requires the corresponding logit to be `+∞`. Real models can't do this, so the gradient on the correct class is *always* pushing the logit higher with no upper bound. Label smoothing caps the reward: the model is "done" once it places `1 − α + α/K` on the right class. This:

1. Prevents over-confident logits that hurt calibration.
2. Improves generalisation by ~1% top-1 accuracy at ImageNet scale.
3. Makes the loss landscape smoother near the optimum.

Typical setting: `α = 0.1`. Used by GPT-2, T5, and most large-scale classifiers.
