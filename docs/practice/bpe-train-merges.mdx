from collections import Counter


def train_bpe(corpus, num_merges):
    merges = []
    for _ in range(num_merges):
        # 1. Count every adjacent pair across all words.
        counts = Counter()
        for word in corpus:
            for i in range(len(word) - 1):
                counts[(word[i], word[i + 1])] += 1
        if not counts:
            break

        # 2. Pick the most-frequent pair, lex tie-break.
        best = max(counts.items(), key=lambda kv: (kv[1], -1 * 0, *(kv[0])))  # placeholder
        # The above is a hack; cleaner: sort candidates by (count, -lex-rank).
        # We use: max count then min lex.
        max_count = max(counts.values())
        best_pair = min(p for p, c in counts.items() if c == max_count)
        merges.append(best_pair)

        # 3. Apply the merge to every word.
        a, b = best_pair
        new_corpus = []
        for word in corpus:
            new_word, i = [], 0
            while i < len(word):
                if i + 1 < len(word) and word[i] == a and word[i + 1] == b:
                    new_word.append(a + b)
                    i += 2
                else:
                    new_word.append(word[i])
                    i += 1
            new_corpus.append(new_word)
        corpus = new_corpus

    return merges

Train Byte-Pair Encoding from a corpus — the algorithm GPT-2 used to build its 50,257-token vocabulary, and the canonical "implement BPE training" interview question.

The algorithm in plain English:

1. Start with each word as a list of single base tokens (characters or bytes).
2. Count every **adjacent pair** of tokens across the corpus.
3. Find the **most frequent** pair. Break ties lexicographically (sort by the `(a, b)` tuple).
4. Merge that pair everywhere — replace `[a, b]` with `[a + b]` in every word.
5. Record the merge. Repeat for `num_merges` iterations.

Two subtleties the harness probes:

1. **Lex tie-breaking.** The most-frequent pair is often tied. Real BPE implementations either freeze a deterministic tie-break (lex order, what we use) or accept implementation-dependent results (which makes them un-reproducible). The harness uses lex tie-breaking and tests it explicitly.
2. **Single-pass merge application.** Walking the word and checking `word[i:i+2] == (a, b)` is the right way. A `str.replace`-based shortcut breaks when merged tokens contain characters that re-appear elsewhere — `replace` operates at the substring level and ignores token boundaries.

This is the *companion* to the existing `bpe-apply-merges` problem. Together they form a complete BPE: train on a corpus, freeze the merge list, then apply that list to encode any new text.
