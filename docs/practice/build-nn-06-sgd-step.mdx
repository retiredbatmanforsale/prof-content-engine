import numpy as np

def sgd_step(params, grads, lr):
    if len(params) != len(grads):
        raise ValueError(f"len(params)={len(params)} != len(grads)={len(grads)}")
    for p, g in zip(params, grads):
        if p.shape != g.shape:
            raise ValueError(f"shape mismatch: param {p.shape} vs grad {g.shape}")
        p -= lr * g     # in-place subtraction
`}

>

Stochastic gradient descent is the simplest optimiser there is — but writing it correctly trips up more people than you'd expect. The two failure modes the harness probes for:

1. **In-place mutation.** The function returns nothing — it updates the parameters by mutating them. If you write `p = p - lr * g` inside the loop, you've created a new local array and the caller's reference still points to the old values. Use `p -= lr * g` (or `np.subtract(p, lr * g, out=p)`) to modify the underlying buffer.
2. **Validation.** `len(params) != len(grads)` should raise (otherwise `zip` silently truncates and some parameters never update — a very ugly real-world bug). Same for per-element shape mismatches.

In future problems we'll layer momentum, weight decay, and adaptive learning rates on top — but everything starts from this two-line update.
