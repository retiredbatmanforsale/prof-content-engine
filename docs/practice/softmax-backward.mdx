import numpy as np

def softmax_backward(grad_out, softmax_out):
    y = softmax_out
    # The closed-form Jacobian-vector product, no (V, V) matrix materialised.
    return y * (grad_out - np.dot(y, grad_out))

Backward pass through softmax — done the right way. The Jacobian of softmax has the form:

`J[i, j] = y_i · (δ_ij − y_j)`

But materialising it as a `(V, V)` matrix is a non-starter for any real-world `V`. Instead, the **Jacobian-vector product** simplifies to:

`dx = y · (grad_out − ⟨y, grad_out⟩)`

Two operations: one dot product (the y-weighted average of `grad_out`), one elementwise multiply. Total cost `O(V)`.

The diagnostic test the harness probes: with `grad_out = constant`, the answer should be exactly 0. Why? Because softmax is **shift-invariant** in its input — adding a constant to every logit doesn't change the output. So any input perturbation in the constant direction has zero effect on the output, and the gradient must be zero. Get the formula wrong (e.g., forget to subtract `dot(y, grad_out)`) and this test catches it immediately.

This Jacobian-vector form is what every framework uses. PyTorch, JAX, TensorFlow all collapse the softmax Jacobian to this O(V) computation under the hood — the (V, V) matrix never exists.
