import numpy as np

def mse_loss(pred, target):
    return np.mean((pred - target) ** 2)


def mse_loss_backward(pred, target):
    N = pred.size  # divide by total element count, matching the .mean() in forward
    return 2.0 * (pred - target) / N
`}

>

Mean squared error is the regression workhorse — used wherever you're predicting continuous values: pixel intensities in autoencoders, value functions in RL, denoising scores in diffusion models. Implement both halves.

The math:

`forward:  L = (1/N) · sum_i (pred_i - target_i)^2`

`backward: dL / d(pred_i) = 2 · (pred_i - target_i) / N`

The single subtlety the harness probes for: **what is `N`?** For 1-D inputs it's the obvious thing — `len(pred)`. For 2-D inputs (batch × features) it's `pred.size`, the total element count. The "loss is averaged over ALL elements" choice matches PyTorch's `MSELoss(reduction='mean')` default and keeps the gradient magnitude scale-invariant to batch size. If you divide by the wrong N, the analytic gradient won't match the finite-difference reference — and the harness will catch it.
