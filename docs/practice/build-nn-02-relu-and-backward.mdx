import numpy as np

def relu(x):
    return np.maximum(0, x)


def relu_backward(grad_out, x):
    return grad_out * (x > 0)
`}

ReLU — `max(0, x)` — is the workhorse non-linearity that made deep networks tractable. It's cheap, doesn't saturate on the positive side, and has a gradient of exactly 1 where it's active. Implement both halves: the forward pass and the backward.

The math:

`forward:  y = max(0, x)`

`backward: dx = dy * 1[x > 0]`

Two subtleties the harness checks:

1. **The sub-gradient at x = 0.** ReLU isn't differentiable at exactly 0; we pick the sub-gradient = 0 by convention. (PyTorch, JAX, and TensorFlow all pick this.) The "dying ReLU" problem flows directly from this choice — once a unit's pre-activation goes ≤ 0 across all training data, its gradient is 0 forever and the unit never recovers.
2. **Backward takes the input, not the output.** Many candidates accidentally write `relu_backward(grad_out, relu_out)` instead of `relu_backward(grad_out, x)`. Both happen to be equivalent for ReLU specifically (since `relu_out > 0 ⟺ x > 0`), but the convention this course uses is to pass the input — that generalises to other activations where it matters.
