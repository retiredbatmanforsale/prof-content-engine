@__test__("sigmoid_backward: matches finite-difference numerical gradient")
def t():
    rng = np.random.default_rng(2)
    x = rng.standard_normal(15)
    s = sigmoid(x)

    # Pretend the loss is sum(sigmoid(x)). Then d_loss/d_x_i = sigmoid'(x_i).
    grad_out = np.ones_like(s)
    analytic = sigmoid_backward(grad_out, s)

    eps = 1e-5
    numeric = np.zeros_like(x)
    for i in range(len(x)):
        h = x.copy(); h[i] += eps
        l_plus = sigmoid(h).sum()
        h[i] -= 2 * eps
        l_minus = sigmoid(h).sum()
        numeric[i] = (l_plus - l_minus) / (2 * eps)

    assert np.allclose(analytic, numeric, atol=1e-4), \\
        f"max diff {np.max(np.abs(analytic - numeric)):.2e}"

import numpy as np

def sigmoid(x):
    # Stable form: branch on the sign of x so the exp argument is always negative.
    out = np.empty_like(x, dtype=float)
    pos = x >= 0
    out[pos] = 1.0 / (1.0 + np.exp(-x[pos]))
    e = np.exp(x[~pos])
    out[~pos] = e / (1.0 + e)
    return out


def sigmoid_backward(grad_out, sigmoid_out):
    return grad_out * sigmoid_out * (1.0 - sigmoid_out)
`}

Sigmoid — `1 / (1 + exp(-x))` — predates the deep learning era and is still the activation of choice when you need a probability between 0 and 1. Implement both halves: the forward pass and the backward.

Two interview-grade subtleties the harness probes for:

1. **Numerical stability.** The textbook form overflows for very negative inputs because `exp(-x)` blows up. The fix is one of two equivalent forms — pick the one whose `exp(...)` argument is always non-positive. (`exp(x) / (1 + exp(x))` for `x < 0`, the original for `x >= 0`. Both equal sigmoid; only one stays finite.)
2. **The `s * (1 - s)` shortcut.** The derivative of sigmoid w.r.t. its input is `sigmoid(x) * (1 - sigmoid(x))`. If you save the forward output, the backward is one cheap multiply — no extra `exp` calls. This is why every framework's API takes the saved output, not the input.

The saturation in the gradient — `s * (1 - s) ≈ 0` at the tails — is exactly the "vanishing gradient through deep sigmoid stacks" problem that ReLU was invented to fix.
