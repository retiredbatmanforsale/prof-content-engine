import numpy as np

def _adam_only(p, g, m, v, t, lr, beta1, beta2, eps):
    # Reference: plain Adam (no weight decay).
    m_new = beta1 * m + (1 - beta1) * g
    v_new = beta2 * v + (1 - beta2) * g * g
    m_hat = m_new / (1 - beta1 ** t)
    v_hat = v_new / (1 - beta2 ** t)
    return p - lr * m_hat / (np.sqrt(v_hat) + eps), m_new, v_new

import numpy as np

def adamw_step(params, grads, m, v, t, lr, beta1, beta2, eps, weight_decay):
    bc1 = 1.0 - beta1 ** t
    bc2 = 1.0 - beta2 ** t
    for p, g, m_p, v_p in zip(params, grads, m, v):
        # Update moments — note: weight decay is NOT added to g here.
        m_p *= beta1; m_p += (1.0 - beta1) * g
        v_p *= beta2; v_p += (1.0 - beta2) * g * g
        m_hat = m_p / bc1
        v_hat = v_p / bc2
        # Combined update: Adam term + DECOUPLED weight decay term.
        p -= lr * (m_hat / (np.sqrt(v_hat) + eps) + weight_decay * p)
`}

>

AdamW — Adam with **decoupled weight decay**. The optimiser GPT-2, GPT-3, Llama, and most modern transformer training runs use. The single-line change from Adam is what makes weight decay actually work as intended.

The update:

`p ← p − lr · (m̂ / (√v̂ + ε) + weight_decay · p)`

The two terms in the parentheses do different things:
- `m̂ / (√v̂ + ε)` — the standard Adam adaptive update, scaling each parameter's step by its own gradient history.
- `weight_decay · p` — a constant L2-style shrinkage applied **outside** the adaptive scaling.

The wrong way (Adam-with-L2): add `weight_decay · p` to `g` before computing `m, v`. Then the decay flows through `m̂ / √v̂` and gets *scaled* by the per-parameter gradient history. Parameters with large gradients (which need MORE regularisation) get LESS decay. Parameters with small gradients get more. That's backwards.

[Loshchilov & Hutter (2017)](https://arxiv.org/abs/1711.05101) showed that decoupling decay restores the regularisation behaviour everyone expected from Adam, and it has been the default for transformer training since.

The diagnostic test the harness probes: after one step, `m` should equal `(1 − β₁) · g`, NOT `(1 − β₁) · (g + wd · p)`. Get the decoupling right and that follows automatically.
