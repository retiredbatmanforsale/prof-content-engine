import numpy as np

# (helpers omitted — same as starter)

def train_xor(seed=0, hidden=4, lr=0.5, steps=5000):
    rng = np.random.default_rng(seed)
    X = np.array([[0., 0.], [0., 1.], [1., 0.], [1., 1.]])
    Y = np.array([[0.], [1.], [1.], [0.]])
    W1 = rng.standard_normal((2, hidden)) * 0.5; b1 = np.zeros(hidden)
    W2 = rng.standard_normal((hidden, 1)) * 0.5; b2 = np.zeros(1)

    for _ in range(steps):
        # forward
        z1 = linear_forward(X, W1, b1); a1 = relu(z1)
        z2 = linear_forward(a1, W2, b2); y = sigmoid(z2)
        # backward
        dy = mse_loss_backward(y, Y)
        dz2 = sigmoid_backward(dy, y)
        da1, dW2, db2 = linear_backward(dz2, a1, W2)
        dz1 = relu_backward(da1, z1)
        _, dW1, db1 = linear_backward(dz1, X, W1)
        # update
        sgd_step([W1, b1, W2, b2], [dW1, db1, dW2, db2], lr)

    return float(mse_loss(y, Y))
`}

This is the capstone of the NN series. Wire together every function you've built so far — `linear_forward`, `relu`, `sigmoid`, `mse_loss`, the three backwards, and `sgd_step` — into a complete training loop. Train a tiny 2 → 4 → 1 MLP on XOR. After 5000 full-batch updates with `lr=0.5`, the MSE loss should land below 0.1.

The architecture and helpers are provided. Your job is the **training loop body**: forward, backward, update, repeat.

Why XOR? It's the smallest non-linearly-separable problem — a single linear layer cannot solve it. A 2-layer MLP with a non-linearity in the middle can. If your training loop converges on XOR, every component is wired correctly: the gradients flow, the optimiser updates in place, the loss decreases.

If you've shipped problems 01–06 and the loss in this problem doesn't drop, the bug is almost always in the backward chain — re-derive each step's `grad_*` shape on paper before running.
