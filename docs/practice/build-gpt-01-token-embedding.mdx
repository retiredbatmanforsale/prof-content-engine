The first thing every transformer touches: turning a sequence of integer token ids into a sequence of learnable vectors. About a third of GPT-2's 124M parameters live in this single embedding table.

The mechanic is the simplest in the model — a row lookup. Given:

`idx ∈ ℤ^(B × T)` — token ids in `[0, V)`

`weight ∈ ℝ^(V × C)` — the embedding table

Return `(B, T, C)` where `out[b, t] = weight[idx[b, t]]`.

The diagnostic test the harness runs: token id `2` appearing three times in a sequence must produce the **exact same** 768-dim vector at all three positions. That's only true if you use the lookup form — a manual loop or a one-hot multiplication would also satisfy it, but you'd be wasting compute.

NumPy gives you this in one character of code. The point of the problem is to know that you don't need to write a for-loop over the batch.
