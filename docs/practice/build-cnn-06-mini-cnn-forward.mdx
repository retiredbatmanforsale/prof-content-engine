import numpy as np

def mini_cnn_forward(x, W_conv, W_fc):
    h = conv2d_naive(x, W_conv)        # (C_out, 6, 6)
    h = relu(h)                         # ReLU non-linearity
    h = max_pool_2d(h, 2, 2)            # (C_out, 3, 3)
    h = h.reshape(-1)                   # flatten to (C_out * 9,)
    logits = h @ W_fc                   # (2,) — 2-class logits
    return softmax(logits)              # (2,) — probabilities

The CNN series capstone. Wire your earlier conv (`build-cnn-01`), ReLU, max pool (`build-cnn-03`), and a softmax classifier into a complete forward pass for a tiny 2-class image classifier.

The architecture:

`(1, 8, 8) image  →  conv 3×3 (C_out filters)  →  ReLU  →  2×2 max pool stride 2`

`              →  flatten  →  Linear(C_out·9 → 2)  →  softmax  →  (2,) probabilities`

The shape walk: 8×8 → 6×6 (conv with kH=3, no pad) → 6×6 (ReLU preserves shape) → 3×3 (pool kernel=stride=2) → flatten to `C_out·9` → matmul with `W_fc: (C_out·9, 2)` → softmax over 2 classes.

Helpers are provided in the starter. Your job: six lines wiring the chain, with the right shapes flowing through.

The diagnostic test reconstructs the chain explicitly and compares — if your forward differs by more than numerical precision, the chain is wired wrong. The "argmax flips when fc columns flip" test is a sanity check that the classification head actually depends on `W_fc`.

This is the capstone of the build-a-CNN series. Once you've shipped this, you can read [LeCun 1998 (LeNet-5)](http://yann.lecun.com/exdb/publis/pdf/lecun-98.pdf) and recognise every piece — same building blocks, more layers and channels.
