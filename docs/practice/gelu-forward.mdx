GELU — Gaussian Error Linear Unit. The smooth, probabilistic cousin of ReLU that GPT-2, BERT, and most transformer-era architectures adopted.

The exact GELU is `x · Φ(x)` (where Φ is the standard normal CDF). The exact form is slow to compute, so OpenAI introduced the **tanh approximation** in the GPT-2 codebase:

`GELU(x) ≈ 0.5 · x · (1 + tanh(sqrt(2/π) · (x + 0.044715 · x³)))`

Within `1e-4` of the exact form across the relevant input range, ~3× faster to compute. PyTorch's `nn.GELU(approximate="tanh")` uses this exact formula, as do nanoGPT and the original OpenAI GPT-2 codebase.

Why GPT-2 prefers GELU over ReLU:

1. **Smooth at zero.** ReLU's gradient jumps from 0 to 1 at the kink; GELU's gradient is continuous. Better-behaved with adaptive optimisers like Adam.
2. **Some negative input gets through.** ReLU zeros all `x ≤ 0`; GELU lets a small fraction of moderately-negative values through, avoiding the "dying ReLU" problem.

Modern alternatives (SwiGLU, GeGLU) further refine this — but GELU is still the default in ~70% of public 2024-era transformers.
