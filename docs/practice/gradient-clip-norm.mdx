import numpy as np

def clip_grad_norm(grads, max_norm):
    # Total L2 norm across the flattened concatenation of all tensors.
    sq = sum(float((g ** 2).sum()) for g in grads)
    total = float(np.sqrt(sq))
    if total > max_norm:
        scale = max_norm / total
        for g in grads:
            g *= scale  # in-place
    return total
`}

Gradient clipping by global L2 norm — the single line of code that prevents most exploding-gradient training crashes. Used by GPT-2, GPT-3, and basically every transformer training run.

The math:

`total = sqrt(sum_i ||g_i||_2^2)`

`if total > max_norm:  g_i ← g_i · (max_norm / total) for all i`

The two correct properties the harness checks:

1. **Direction preserved.** Clipping is a pure scalar shrink — every component scales by the same factor, so the gradient vector points the same way. If you accidentally clip per-tensor instead of globally, the joint direction rotates and you've broken the optimisation step.
2. **In-place mutation.** Same reason as `sgd_step` — the caller passes in a list of arrays expecting them to be modified. Use `g *= scale`, not `g = g * scale`.

The return value (original norm) is what you log at training time. A healthy run shows the norm hovering well below `max_norm`; spikes past `max_norm` mean you just clipped, and a sustained climb is the early warning of training-loss divergence.
