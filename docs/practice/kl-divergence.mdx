import numpy as np

def kl_divergence(p, q):
    # Mask: only sum positions where p > 0. The 0*log(0) convention is enforced
    # by zeroing the contribution at those positions.
    mask = p > 0
    return float(np.sum(p[mask] * np.log(p[mask] / q[mask])))
`}

KL divergence — the asymmetric "distance" between two probability distributions, in nats. Foundation of cross-entropy, ELBO, label smoothing, knowledge distillation, RLHF, and most information-theoretic objectives in modern ML.

The formula:

`KL(p || q) = sum_i p_i · log(p_i / q_i)`

Three properties the harness checks:

1. **`KL(p || p) = 0`** — zero distance from a distribution to itself.
2. **Non-negativity** — `KL ≥ 0` for any valid `p, q` (with equality iff `p = q`). This is Gibbs' inequality.
3. **Asymmetry** — `KL(p || q) ≠ KL(q || p)` in general. That's why it's called a *divergence*, not a distance. The first argument is the "true" distribution, the second is the "approximate" one.

The single subtlety: **the `0 · log(0) = 0` convention.** Mathematically `log(0) = -∞`, but the product is the limit of `x · log(x)` as `x → 0`, which is `0`. Implementation-wise: mask out `p_i = 0` before taking the log. Don't add `eps` inside the log — it biases the answer.
