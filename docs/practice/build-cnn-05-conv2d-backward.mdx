import numpy as np

def conv2d_backward(grad_out, x, W):
    C_in, H, Wd = x.shape
    C_out, _, kH, kW = W.shape
    out_H, out_W = grad_out.shape[1], grad_out.shape[2]
    dx = np.zeros_like(x)
    dW = np.zeros_like(W)
    for o in range(C_out):
        for i in range(out_H):
            for j in range(out_W):
                g = grad_out[o, i, j]
                # Each output cell (o,i,j) was: sum over (c_in,kh,kw) of x[c_in,i+kh,j+kw] * W[o,c_in,kh,kw]
                # So dW[o,:,:,:] += g * x[:, i:i+kH, j:j+kW]
                dW[o] += g * x[:, i:i+kH, j:j+kW]
                # And dx[:, i:i+kH, j:j+kW] += g * W[o,:,:,:]
                dx[:, i:i+kH, j:j+kW] += g * W[o]
    return dx, dW
`}

    "dW accumulation: dW[o, :, :, :] += g * x[:, i:i+kH, j:j+kW]. Each output cell sees a (C_in, kH, kW) patch; multiply by the upstream grad and add to that kernel's gradient.",

  ]}
>

The backward pass through `conv2d_naive` from `build-cnn-01`. Two gradients, one elegant pattern: walk the same triple loop as the forward pass and accumulate.

The math from the chain rule. Forward says:

`out[o, i, j] = sum over (c_in, kh, kw): x[c_in, i+kh, j+kw] * W[o, c_in, kh, kw]`

Differentiate w.r.t. each kernel weight:

`dW[o, c_in, kh, kw] = sum over (i, j): grad_out[o, i, j] * x[c_in, i+kh, j+kw]`

Differentiate w.r.t. each input pixel:

`dx[c_in, h, w] = sum over outputs that touched (c_in, h, w): grad_out[o, i, j] * W[o, c_in, kh, kw]`

Implement both with the **same iteration as the forward**: for each output position, take the upstream gradient `g` and accumulate `dW[o] += g * x_patch` and `dx_patch += g * W[o]`.

The diagnostic tests are finite-difference checks — both `dx` and `dW` cross-validated against numerical gradients of `conv2d_naive(x, W).sum()`. If your derivation is wrong, the differences will be order-of-magnitude visible, not subtle.
