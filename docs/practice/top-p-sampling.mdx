import numpy as np

def top_p_sample(probs, p, rng):
    # Sort descending.
    order = np.argsort(probs)[::-1]
    sorted_probs = probs[order]
    cumulative = np.cumsum(sorted_probs)

    # Smallest prefix whose cumulative ≥ p — but always include at least one.
    keep = np.searchsorted(cumulative, p) + 1
    keep = max(keep, 1)

    nucleus = sorted_probs[:keep]
    nucleus = nucleus / nucleus.sum()

    pick = rng.choice(keep, p=nucleus)
    return int(order[pick])
`}

Top-p (nucleus) sampling is the default decoding strategy for modern LLMs. It strikes a balance between greedy decoding (deterministic, often boring) and full-distribution sampling (creative, occasionally bizarre).

The idea: at each step, only sample from the smallest set of candidate tokens whose probabilities add up to at least `p`. The "nucleus" adapts to the model's confidence — when the model is confident, the nucleus is one or two tokens; when it's uncertain, the nucleus expands.

The algorithm in plain English:

1. Sort token probabilities from highest to lowest.
2. Take the smallest prefix whose cumulative probability reaches `p`. Always keep at least one token (the argmax) — even if it alone already exceeds `p`.
3. Renormalize that prefix to a valid distribution.
4. Sample one token from the renormalized prefix.
5. Map back to the original index.

The harness fixes the RNG with a seed, so your function must use **only** the `rng` argument for randomness — no `np.random.choice(...)` (which uses global state). That's a real-world correctness requirement for reproducible inference.
