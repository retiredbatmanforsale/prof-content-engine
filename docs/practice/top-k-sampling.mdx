import numpy as np

def top_k_sample(probs, k, rng):
    V = len(probs)
    k = min(k, V)
    # Indices of the top-k probs (any order is fine; argpartition is fastest).
    top_idx = np.argpartition(probs, -k)[-k:]
    kept = probs[top_idx]
    kept = kept / kept.sum()
    pick = rng.choice(k, p=kept)
    return int(top_idx[pick])
`}

Top-k sampling — pick the `k` highest-probability tokens, renormalise their probabilities, sample one. The fixed-cardinality alternative to top-p (nucleus) sampling.

The algorithm:

1. Find the indices of the `k` largest entries in `probs`.
2. Renormalise those `k` probabilities to sum to 1.
3. Sample one position from the renormalised distribution.
4. Return the corresponding original-vocabulary index.

Trade-offs vs top-p:

- **Top-k** is simpler to reason about — you always sample from exactly `k` candidates, regardless of model confidence. Fixed compute, fixed cardinality.
- **Top-p** adapts to confidence — when the model is sure, the nucleus is 1-2 tokens; when uncertain, 50+. Often better on LM benchmarks but harder to debug.

GPT-2's typical inference setting was `top_k = 40`. Most production LLMs now default to `top_p ≈ 0.9` instead, but `top_k` survives as a fallback knob and in inference engines that prefer fixed-cardinality sampling for batching reasons.

The harness uses fixed RNG seeds — your function must use **only** the `rng` argument for randomness, never `np.random` global state.
