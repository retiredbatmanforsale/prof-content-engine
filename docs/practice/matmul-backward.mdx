import numpy as np

def matmul_backward(grad_out, A, B):
    dA = grad_out @ B.T   # (M, N) @ (N, K) -> (M, K)
    dB = A.T @ grad_out   # (K, M) @ (M, N) -> (K, N)
    return dA, dB
`}
  hints={[
    "Two lines: dA = grad_out @ B.T; dB = A.T @ grad_out. The shapes alone force this — only one way to multiply (M,N), (M,K), (K,N), (K,N) such that the answers come out (M,K) and (K,N).",
    "Why those expressions? Forward is C = A @ B. The element C[i,j] = sum_k A[i,k] * B[k,j]. So d(C[i,j]) / d(A[i,k]) = B[k,j] — and the gradient at A[i,k] is sum_j grad_out[i,j] * B[k,j], which is exactly the (i, k) entry of grad_out @ B.T.",
    "Sanity check by transposing both sides: if forward is (A @ B).T = B.T @ A.T, then dB^T = grad_out.T @ A and dA^T = B @ grad_out.T. Take the transpose and you get the same answers — symmetry confirms the formulas.",

>

The backward pass for plain matrix multiplication. Underneath every linear layer, attention head, and convolution-via-im2col is exactly this two-line gradient.

Forward: `C = A @ B` with `A: (M, K)`, `B: (K, N)`, `C: (M, N)`.

Backward, given `grad_out = ∂L/∂C` of shape `(M, N)`:

`dA = grad_out @ B.T          # shape (M, K)`

`dB = A.T @ grad_out          # shape (K, N)`

The shapes alone force the formulas — only one way to multiply the available matrices to land on `(M, K)` and `(K, N)`. Get the order or transposes wrong and the matmul fails entirely.

The harness's diagnostic test is finite-difference cross-validation — perturb each entry of `A` (then `B`) by a small epsilon and check that the analytic gradient matches the numerical one within `1e-3`. If your `dA` is `B @ grad_out.T` instead of `grad_out @ B.T` (a common transpose flip), this test catches it.

This is the same pattern as `build-nn-05-linear-backward`'s dx and dW — Linear is a thin wrapper around matmul, with an extra bias term.
