import numpy as np

def focal_loss(logits, target, gamma=2.0, alpha=1.0):
    # numerically-stable softmax
    z = logits - logits.max()
    e = np.exp(z)
    probs = e / e.sum()
    p_t = float(probs[target])
    # Clip p_t to avoid log(0) — the harness inputs don't trigger this but
    # production logits can produce p_t=0 in fp16.
    p_t = max(p_t, 1e-30)
    return float(-alpha * (1.0 - p_t) ** gamma * np.log(p_t))

Focal loss — introduced by [Lin et al. (2017)](https://arxiv.org/abs/1708.02002) for RetinaNet object detection. The training problem they solved: 99% of object-detection candidate boxes are "easy negatives" — clearly background — and they swamp the loss, drowning out gradients from the rare hard examples.

The fix: down-weight examples where the model is already correct.

`L_focal = −α · (1 − p_t)^γ · log(p_t)`

The new factor `(1 − p_t)^γ` is what does the work. Three regimes:

- **`p_t ≈ 1`** (confident-correct): `(1 − p_t)^γ → 0`, loss is crushed near zero. The model "moves on" from solved examples.
- **`p_t ≈ 0`** (confident-wrong): `(1 − p_t)^γ → 1`, loss looks like normal CE but with the `α` reweight.
- **`p_t ≈ 0.5`** (uncertain): `(1 − p_t)^γ ≈ 0.5^γ`, loss is moderately reduced.

Hyperparameters: `γ = 2.0` is the canonical RetinaNet setting; `α` is per-class balancing (`α = 0.25` for the rare class in binary detection). When `γ = 0`, focal loss reduces to alpha-weighted cross-entropy — the harness's "reduces to CE" test verifies this.

Used in object detection (RetinaNet, YOLO derivatives) and any classification problem with severe class imbalance.
