# Practice

LeetCode for ML internals. Each problem ships with a hidden test harness that runs in your browser via Pyodide — no backend, no submission queue, no rate limit. Your code never leaves the tab.

The first five problems target the operations interviewers actually ask about: softmax, attention, BPE, sampling, and the cross-entropy gradient. Each one cross-links back to the lesson that introduces the concept.

## How it works

Each problem page has:

- A statement explaining what you're implementing and why.
- A starter code skeleton — read the docstring, fill in the body.
- **Run tests** — your code executes in Pyodide alongside a hidden test harness. You get a per-test pass/fail panel with timing and (on failure) the assertion message.
- Progressive **hints** if you're stuck.
- A **Reveal solution** button that loads the reference implementation into the editor so you can run it and compare.

The runtime takes 5–10 seconds to spin up the first time per session, then runs are instant.

## Why these five

These are the operations every transformer touches on every forward pass. If you can implement all five from a blank file, numerical stability included, you can debug almost any production LLM bug.

They also happen to be the most-asked ML internals questions in interviews — the kind where "I sort of remember the formula" doesn't survive a 25-minute coding round.
