  solution={`
import numpy as np

def knn_classify(x_query, X_train, y_train, k):
    # 1. Euclidean distances (vectorised).
    diffs = X_train - x_query
    dists = np.sqrt((diffs ** 2).sum(axis=1))
    # 2. Indices of k smallest distances.
    nn_idx = np.argpartition(dists, k - 1)[:k]
    nn_labels = y_train[nn_idx]
    # 3. Majority vote with smallest-label tie-breaking via np.bincount.
    counts = np.bincount(nn_labels)
    return int(np.argmax(counts))   # ties resolve to the smallest index
`}

K-nearest-neighbours classification — the simplest non-parametric classifier and the warm-up problem in every classical-ML interview.

The algorithm is three lines of vectorised numpy:

1. Euclidean distance from `x_query` to every row of `X_train`.
2. The `k` indices with the smallest distances.
3. Majority vote among the corresponding labels.

Two subtleties the harness probes for:

1. **Tie-breaking is deterministic.** When two or more classes tie for the most votes among the k neighbours, return the **smallest class label**. Using `np.bincount` + `np.argmax` gives this for free — `argmax` returns the smallest index when bins are tied. If you implement majority voting with a Counter or hashmap, you'll need explicit tie-breaking logic.
2. **Vectorise the distance calculation.** `np.linalg.norm(X_train - x_query, axis=1)` works, or `np.sqrt(((X_train - x_query) ** 2).sum(axis=1))`. A Python `for i in range(N)` loop will work for the harness's small N but is the wrong instinct for production.

For larger datasets, k-d trees and ball trees beat the naive O(N·D) distance computation — but those don't fit in 25 lines.
