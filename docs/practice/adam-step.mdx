import numpy as np

def adam_step(params, grads, m, v, t, lr, beta1, beta2, eps):
    bc1 = 1.0 - beta1 ** t        # bias-correction denominators
    bc2 = 1.0 - beta2 ** t
    for p, g, m_p, v_p in zip(params, grads, m, v):
        m_p *= beta1; m_p += (1.0 - beta1) * g
        v_p *= beta2; v_p += (1.0 - beta2) * g * g
        m_hat = m_p / bc1
        v_hat = v_p / bc2
        p -= lr * m_hat / (np.sqrt(v_hat) + eps)
`}

Adam — the Adaptive Moment Estimation optimiser. The default for transformer training before AdamW (which differs by one weight-decay term). Knowing it cold is table-stakes for any ML interview.

The update rule, applied per parameter:

`m ← β₁ m + (1 − β₁) g            (first-moment EMA)`

`v ← β₂ v + (1 − β₂) g²           (second-moment EMA)`

`m̂ = m / (1 − β₁ᵗ)               (bias-corrected first moment)`

`v̂ = v / (1 − β₂ᵗ)               (bias-corrected second moment)`

`p ← p − lr · m̂ / (√v̂ + ε)`

Defaults from the original paper: `lr=0.001, β₁=0.9, β₂=0.999, ε=1e-8`.

Two subtleties the harness probes:

1. **Bias correction.** `(1 − β^t)` denominators compensate for `m, v` being initialised to zero. At `t=1`, `m_hat = g` exactly (the `(1-β)*g` numerator and `(1-β)` denominator cancel) — so the very first step is roughly `lr · sign(g)`, regardless of `β`. This is what lets Adam not stall at step 1.
2. **In-place update of three buffers.** `m` and `v` persist across steps as optimiser state; if you rebind them inside the loop, the optimiser silently resets every step.
