import numpy as np

def max_pool_2d(x, kernel_size, stride):
    C, H, W = x.shape
    k, s = kernel_size, stride
    out_H = (H - k) // s + 1
    out_W = (W - k) // s + 1
    out = np.zeros((C, out_H, out_W))
    for c in range(C):
        for i in range(out_H):
            for j in range(out_W):
                out[c, i, j] = x[c, i*s:i*s+k, j*s:j*s+k].max()
    return out
`}

Max pooling — the simplest downsampling op in a CNN. For each channel, slide a `k × k` window with step `stride` and take the maximum within. The output is `(C, (H-k)//s + 1, (W-k)//s + 1)`.

Why max specifically? Two reasons it survived the deep-learning era:

1. **Translation invariance.** A small shift in the input that keeps the maximum activation in the same window produces the same output. Useful for "is there a vertical edge anywhere in this region?"-style features.
2. **Sparse gradients.** During backward, only the position that had the max gets a non-zero gradient. The rest get zero — pooling acts as a routing operation, not a smoothing one.

The classic config is `k = 2, stride = 2` — halves H and W, keeps the strongest activation in each `2 × 2` block. That's what the diagnostic test checks.

`build-cnn-04` will pair this with average pooling. Both are still useful in 2026 CNNs, even after global pooling and learned downsampling have replaced them in most ImageNet-scale architectures.
